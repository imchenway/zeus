# Zeus 任务与会话语境

Zeus 用任务表达待完成的工作，用会话承载人与 AI 的持续交互；两者相关联，但生命周期和操作责任不同。

## Language

**任务**：
需要被跟踪、验收和归档的工作目标；任务只记录工作状态，不直接拥有 AI 执行控制。
_Avoid_: Runtime、会话

**会话**：
围绕一个项目持续保存用户消息、AI 输出和执行上下文的交互空间；一个任务可以关联一个或多个会话。
_Avoid_: 任务、Runtime 进程

**会话可继续性**：
用户能否在原有上下文中继续发送消息的产品能力；底层会话的存储或运行状态不是用户术语。
_Avoid_: provider 归档、恢复并继续

**执行轮次**：
会话中一次从用户输入开始、以 AI 完成或被中断结束的执行过程。
_Avoid_: 暂停会话、暂停 Runtime

**完成未读**：
执行轮次已经成功结束，但用户尚未打开对应会话查看结果的状态；失败或中断不属于完成未读。
_Avoid_: 会话就绪、运行完成、通用未读消息

**PLAN 协作模式**：
用户明确要求 Codex 先分析并形成可确认计划的会话提交模式；它逐次随提交记录，不等同于计划步骤进度，也不代表已经授权实施。
_Avoid_: 计划进度、自动执行模式

**计划产物**：
app-server `type: "plan"` item 所承载的持久化 Markdown 计划正文；消息流计划卡片与右侧计划工作区只是同一份内容的两种视图。
_Avoid_: 计划进度、复制出的文件、套餐

**计划实施请求**：
PLAN 执行轮次成功完成并形成非空计划产物后，由 Zeus 持久化生成的独立确认请求；它负责实施、修订或跳过计划，不伪装成 app-server
`request_user_input`。
_Avoid_: 普通用户询问、审批请求、计划产物

**普通用户询问**：
app-server `request_user_input` 发出的阻塞式问题，按 provider 问题与选项原样回答；它与计划实施请求使用相近视觉原语，但协议、生命周期和响应规则独立。
_Avoid_: 计划实施请求、命令审批

**类似命令授权**：
用户接受 provider 明确提出的一条命令规则，使符合该规则的后续命令无需逐次批准；它只覆盖展示给用户的确切规则。
_Avoid_: 本会话允许、允许所有命令、自动审批

**单次允许**：
用户只批准当前一条命令或当前一次文件修改请求；协议分别使用对应请求类型的 `accept` 决策。
_Avoid_: 始终允许、自动审批

**会话文件允许**：
文件审批的 `acceptForSession` 决策；当前上游只记忆本会话已经审批过的文件，新文件仍可能再次申请。界面按 Codex App
显示“允许所有编辑”，但范围说明必须保留这一事实。
_Avoid_: 项目永久授权、允许任意路径、关闭全部审批

**自动模式**：
以项目目录为可写根并按需审批的权限模式；普通项目内编辑通常直接执行，高风险命令、网络或越界访问仍可能申请审批。
_Avoid_: 完全访问、永不审批、只读

**任务首发**：
以任务内容作为首条用户消息发起会话的用户动作；界面名称为“推送到模型”。
_Avoid_: 创建流程、Runtime 启动

**中断当前执行**：
终止会话中正在进行的执行轮次，同时保留会话及其历史，使用户之后仍可继续发起新的执行轮次。
_Avoid_: 暂停任务、停止会话

**MCP 服务标识**：
app-server 在 MCP 启动状态中提供、用于索引单个 MCP 服务状态的名称；它是服务身份标签，不是凭据字段，名称中可以合法包含
`api-key` 等文本。
_Avoid_: API Key、密钥值、provider 负载字段
