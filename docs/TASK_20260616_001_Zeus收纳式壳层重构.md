# TASK_20260616_001 Zeus 收纳式壳层重构

## 1. 任务目标

用户明确要求把 Zeus 从“功能清单平铺页”改成“按真实操作路径递进的产品壳层”。本次目标不是新增后端能力，而是重构桌面端信息架构：

- 左侧一级导航只保留：项目、对话、设置。
- 右侧内容按当前页面唯一目标展示，不再把 Dashboard、Tasks、Runtime、Code Map、Git Diff、Telegram、Settings 全部铺开。
- Runtime、Code Map、Git Diff、Telegram、归档、模板、配置、危险操作全部收纳到抽屉、分类、详情区或确认区。
- 保留隐藏标题栏顶部拖动能力，不回退到只能拖边缘。

## 2. 当前需求涉及的用户操作路径

| 操作路径 | 用户意图 | 新入口 | 首层是否展示 | 收纳位置 |
|---|---|---|---|---|
| 选择/准备项目 | 选择真实本地仓库，确认扫描和 Git 状态 | 项目 | 是 | 项目编辑、配置、归档、图谱、变更进入项目抽屉 |
| 扫描项目/打开图谱 | 从当前项目进入源码理解 | 项目 | 只展示“打开图谱/扫描项目”按钮 | 代码图谱抽屉 |
| 查看代码变更 | 确认当前仓库变更和 Git 风险 | 项目 | 只展示“查看变更”按钮 | 代码变更抽屉，Git 写操作放危险区 |
| 推进任务/对话 | 继续当前任务、输入要求、推送 CLI | 对话 | 是 | Runtime、上下文、代码变更、模板进入对话抽屉 |
| 查看 Runtime 会话/日志 | 判断 AI CLI、会话、日志状态 | 对话 | 否 | 运行环境抽屉 |
| 使用任务模板 | 从模板创建新任务 | 对话 | 否 | 模板抽屉 |
| 修改配置 | 安全修改单个配置分类 | 设置 | 分类导航 + 单分类内容 | 高级项 details，危险项独立危险区 |
| 配置 Telegram | 保存 Token、通知和轮询信息 | 设置 > Telegram | 否 | 单独分类，日志 details |
| 安全/Keychain | 管理外部 Key 和白名单 | 设置 > 安全与 Keychain | 否 | 单独分类，审计 details |
| 发布状态 | 查看签名、公证、Homebrew 与更新等待项 | 设置 > 发布与更新 | 否 | 单独分类，发布详情 details |

## 3. 每个页面唯一核心目标

### 3.1 项目页

唯一目标：选定并准备当前项目。

首层展示：

- 项目列表。
- 当前项目路径、扫描状态、Git 状态、图谱数量。
- 下一步按钮：扫描项目、打开图谱、查看变更。

必须收纳：

- 代码图谱：项目抽屉。
- Git Diff：项目抽屉。
- 项目编辑：项目抽屉。
- 项目配置：项目抽屉。
- 归档项目：项目抽屉。
- Git 写操作：代码变更抽屉内危险区。

### 3.2 对话页

唯一目标：推进当前任务或 CLI 对话。

首层展示：

- 对话/任务列表。
- 当前对话内容、任务事件摘要。
- 输入区。
- 主操作：推送到 CLI 对话、暂停、继续、标记完成、取消任务。

必须收纳：

- Runtime 会话、日志、Adapter、Generic shell：运行环境抽屉。
- Code Map 上下文：上下文抽屉。
- Git Diff：代码变更抽屉。
- 任务模板：模板抽屉。
- 归档任务：左侧归档任务分组。

### 3.3 设置页

唯一目标：安全修改一个配置分类。

首层展示：

- 左侧设置分类。
- 右侧当前分类内容。

固定分类：

- 通用。
- AI CLI / Runtime。
- Telegram。
- 安全与 Keychain。
- Git 确认。
- 发布与更新。
- 缓存与数据。

必须收纳：

- 高级 Runtime 参数：details。
- Telegram 轮询与消息日志：details。
- 安全审计：details。
- 发布详情：details。
- 危险操作：danger-zone。

## 7. 契约变更

### 7.1 导航契约

- `projects`：项目。
- `conversations`：对话。
- `settings`：设置。

旧 hash 兼容迁移：

- `dashboard`、`tasks`、`runtime` -> `conversations`。
- `code-map`、`git-diff` -> `projects`。
- `telegram`、`settings-*` -> `settings`。

### 7.2 页面渲染契约

- 任意时刻只渲染一个一级工作区。
- 项目/对话页采用左列表 + 右详情结构。
- 设置页采用左分类 + 右当前分类结构。
- `DataPanel` 不再作为通用页面平铺原语存在。
- 常驻 `ContextRail` 不再渲染。

### 7.3 安全契约

- Git 写操作仍必须先创建确认，确认后才显示执行入口。
- Generic shell 保留高危短语确认。
- Keychain/API Key/Telegram Token 不回显明文。
- 发布签名/公证未配置时明确显示等待，不伪造成已成功。
