# TASK_20260616_002 Zeus 项目路径去重与一屏布局

## 任务背景

用户反馈：Projects 搜索结果中出现相同项目路径的重复项目，例如 `/Users/david/hypha/zeus` 与 `/Users/david/cckg/tcapp/Back-End/tc-app-core` 各显示两条。用户明确要求：项目路径不允许重复，一个路径只有一个项目。

用户同时指出：当前所有菜单内容都像网页后台一样纵向平铺，正常产品应尽量一屏展示完整，不应把列表、详情、配置、归档等全部往下堆。

## 根因

1. 存储层 `ProjectRepository.create()` 原来每次都插入新项目，没有按 `local_path` 做 active 项目唯一约束。
2. `ProjectRepository.update()` 原来允许把某个项目路径改成另一个 active 项目的路径。
3. Renderer 原来直接渲染 `snapshot.projects`，即使上游返回重复路径，也会按记录数全部展示。
4. Projects 页面原来把搜索、项目列表、详情、配置、归档空面板全部塞在同一个纵向流里，视觉上就是“菜单内容平铺长页面”。

## 设计结论

采用“不破坏历史数据、但 active 视图和后续写入强约束”的方案：

- 存储层创建项目时先规范化路径，若同路径未删除项目已存在，直接返回已有项目，不再新增记录。
- 存储层更新项目时禁止移动到其他未删除项目的路径。
- Local Server 创建/编辑项目时规范化绝对路径并去掉尾部斜杠，但不使用 `realpathSync`，避免 macOS `/var` 被改写为 `/private/var`，保留用户选择路径口径。
- Renderer 对项目列表做二次防御式去重，保证即使旧数据库或导入数据带重复路径，UI 仍只展示一个项目。
- Projects 页面改为一屏工作台：左侧项目搜索与列表，右侧项目详情、编辑和配置。各区域独立滚动，避免整个菜单纵向铺开。
- 归档项目为空时不再渲染空面板；只有存在真实归档项目时才显示归档区域。

## 契约变更

| 契约 | 结论 |
|---|---|
| HTTP API | 路径重复创建不再产生新项目，返回已有项目记录；未新增接口。 |
| DB schema | 未新增表/字段/索引；避免对已有重复历史数据做破坏性迁移。 |
| UI DOM | Projects 页新增 `projects-workbench`、`project-browser`、`project-inspector` 语义类；重复路径只渲染一个项目控制区。 |
| 路径规范化 | 去除尾部斜杠并转绝对路径；不改写 symlink/`/var` 到真实物理路径。 |

## 实施顺序

1. 跑受影响测试 baseline，确认已有 2 个旧 UI 文案断言失败。
2. 先新增红灯测试：存储路径唯一、更新冲突、Projects 一屏布局和 UI 去重。
3. 实现存储层路径规范化、去重和更新冲突保护。
4. 实现 Local Server 路径规范化写入，并避免 `realpathSync` 改写用户路径。
5. 实现 Renderer 防御式去重与 Projects 一屏双栏布局。
6. 修复旧 UI 契约隐藏标题，重跑聚焦测试。
7. 跑完整受影响三文件和类型检查。

## 风险与回滚

| 风险 | 说明 | 回滚 |
|---|---|---|
| 旧数据库中已有重复项目记录 | 本次不做破坏性迁移，UI 和 search/list 会隐藏重复 active 路径；底层旧 row 仍保留审计历史。 | 如需彻底清理，可另做显式数据清理脚本，迁移任务需用户确认。 |
| archived 项目同路径再创建 | 存储层按未删除项目查重，归档项目也会阻止重复创建，符合“一个路径一个项目”。 | 如未来要允许“归档后重新创建”，需改为恢复归档项目而不是新增项目。 |
| 一屏布局在极窄窗口 | 1120px 以下会回退单列，但仍保持内部滚动，不再让整个菜单无限平铺。 | 回滚 `styles.css` 末尾本任务布局覆盖即可。 |
| 仅 SSR/单测验证，未启动 macOS GUI | 本轮遵守非侵入原则，未主动拉起 Electron GUI。 | 如需要真实视觉截图，可在用户确认后启动 app 做浏览/截图验证。 |
