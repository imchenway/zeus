# TASK_20260616_003 Zeus 组件样式对齐

## 1. 任务背景

用户给出 3 张 Codex macOS 设置界面截图，要求“对齐一下这些组件的样式，做成一样的”。截图中的共同特征：

- 设置内容以圆角白色分组承载，组内通过细分割线划分行。
- 每一行左侧是标题和说明，右侧是当前控件或状态。
- select、按钮、switch 都是轻量灰底控件，hover/focus/disabled 状态统一。
- 项目列表以同一套圆角列表卡片展示，每行包含轻量图标、主标题、副标题和右侧操作/状态。

本次不是新增业务能力，也不是回退到后台式卡片堆叠；目标是把现有 Projects / Settings 中漂移的控件样式收敛为一套可复用的 macOS 产品组件词汇。

## 4. 契约变更

### 4.1 组件语义契约

- 设置页当前分类内容必须包含 `native-settings-stack`。
- 每个设置分类内容使用 `native-settings-card` 作为圆角分组外壳。
- 通用设置首层行使用 `native-control-row`、`native-control-copy`、`native-control-slot`。
- 桌面通知开关使用 `native-switch-input`，视觉上与截图中的蓝色/灰色 switch 对齐。
- 项目列表使用 `native-list-card` 和 `native-list-row`，每行包含 `native-folder-icon`、`native-list-main`、`native-list-trailing`。

### 4.2 视觉契约

- 统一使用 `--zeus-native-surface`、`--zeus-native-row-separator`、`--zeus-native-control`、`--zeus-native-accent` 等 token。
- 行组圆角为 14px，行内控件圆角约 9-10px。
- 行间通过 `--zeus-native-row-separator` 细分割线区分，不用重复卡片堆叠。
- hover 只做轻背景变化；focus 使用已有蓝色 focus ring；不新增夸张动画。

### 4.3 信息架构边界

- 仍保持 Projects / Conversations / Settings 三个一级入口。
- Settings 仍只展示一个分类，不恢复所有设置组平铺。
- Projects 仍保持左列表 + 右详情，不恢复旧 Dashboard / DataPanel / ContextRail。

## 6. 实施顺序

1. 读取项目规范、设计上下文和最新 docs，确认当前是 macOS 产品 UI、非营销页。
2. 运行 baseline：`app-shell-layout.test.tsx` + `renderer.test.tsx` 已全绿。
3. 按 TDD 新增 3 个红灯测试，确认失败原因是缺少统一 row-card/list-card/switch 样式契约。
4. 在 `App.tsx` 新增共享组件，并替换 Settings / Projects 相关结构。
5. 在 `styles.css` 新增 Codex macOS 设置组件 token 与控件样式覆盖。
6. 额外运行 `pnpm lint` 发现 `App.tsx` 历史遗留不可达变量/函数；删除已不可达死代码，补回真实可达筛选 UI 与状态展示，避免用 `void` 占位假修复。
7. 运行聚焦测试、受影响数据渲染测试、lint、typecheck 与 desktop build。
8. 更新本文档，记录影响范围、契约、测试矩阵、风险和回滚。

## 7. 风险与回滚

| 风险 | 影响 | 控制 | 回滚 |
|---|---|---|---|
| CSS 覆盖顺序与旧 Apple flat 控件体系冲突 | 某些按钮/列表行仍显示旧边框或阴影 | 新样式块放在文件末尾，专门覆盖 native row/list/card | 回滚 `styles.css` 新增 “Codex macOS 设置组件对齐” 块 |
| Settings 说明文案被旧全局隐藏规则吞掉 | 左标题右说明无法呈现 | 说明使用 `span.native-control-description`，避开全局隐藏 `p/small/em` | 回滚 `NativeControlRow` 结构或调整隐藏规则 |
| 项目列表路径从 `small` 改为 `span` | 视觉与测试可能受影响 | 数据渲染测试已确认路径仍在 SSR 中出现，且重复路径仍隐藏 | 恢复原行结构或保留 span 但移除样式类 |
| 只做静态/单测，未做真实 GUI 截图 | 可能存在像素级偏差 | 遵守非侵入原则；如用户允许，可后续启动 Electron 做截图对照 | 不涉及代码回滚 |
