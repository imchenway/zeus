# TASK_20260616_004 Zeus 开源发布、安装与升级改造

## 1. 任务背景

用户要求 Zeus 作为开源项目完成工程化审查改造，并进一步支持类似 Codex 的新用户安装、升级、应用内检测更新。核心动机不是“把代码推到 GitHub 就结束”，而是让陌生用户能从公开仓库理解、安装、验证和安全升级 Zeus。

本次实现遵循本地优先原则：签名、公证、Homebrew tap token 等外部凭据未配置时，只展示等待项或手动安装路径，不伪造自动更新成功。

## 3. 契约变更

### 3.1 安装与更新

- GitHub Release 应包含：`Zeus-*.dmg`、`Zeus-*.zip`、`Zeus-latest-*.dmg`、`Zeus-latest-*.zip`、`SHA256SUMS`、`install.sh`、`zeus-release-manifest.json`、`homebrew/zeus.rb`。
- `install.sh` 支持 `ZEUS_NON_INTERACTIVE`、`ZEUS_INSTALL_DIR`、`ZEUS_CHANNEL`。
- 应用内“检查更新”读取 Release manifest；未签名/未公证时 `recommendedAction=open_download_page`，不自动下载替换 App。
- 签名且公证都满足时，核心判断允许 `download_and_install`，但本轮下载/安装 API 仍返回预留说明，避免无 UI 确认地替换本机 App。

### 3.2 API

| API | 含义 | 安全口径 |
|---|---|---|
| `GET /api/release/update-status` | 获取当前更新状态 | 不返回签名环境变量或 secret。 |
| `POST /api/release/check-update` | 手动检查更新 | 读取 manifest，可通过测试注入。 |
| `POST /api/release/download-update` | 下载预留 | 未签名/未公证时拒绝静默下载。 |
| `POST /api/release/install-update` | 安装预留 | 未签名/未公证时拒绝自动替换 App。 |

### 3.3 安全

- Renderer 和系统通知 WebSocket token 改为 `sec-websocket-protocol: zeus-token.<base64url(token)>`，不再放 query string。
- Electron Renderer 启用 `sandbox: true`，关闭 Node integration，保留 context isolation。
- Electron Main 拒绝任意 window open、非 Zeus renderer 导航和浏览器权限请求。
- HTML 加 CSP：禁止 object、frame ancestors，仅允许本地 API/WebSocket 连接。

## 6. 风险与回滚

| 风险 | 影响 | 控制 | 回滚 |
|---|---|---|---|
| Electron 42 / Vite 8 升级破坏打包 | 构建或 package:mac 失败 | 保留 package 脚本和 package-mac 测试；发布门禁会真实打包 | 回退 `package.json`、`pnpm-lock.yaml`、`electron-builder.yml` 的版本变更。 |
| 自动更新被误解为已能静默替换 | 用户安装未签名包有安全风险 | API/UI 文案明确 unsigned 只手动安装，download/install 返回预留拒绝 | 移除更新 API/UI，保留 release status。 |
| install.sh 下载 latest alias 缺失 | 新用户安装失败 | verify-release 生成 `Zeus-latest-${arch}.dmg` 与 `SHA256SUMS` | 回退 install.sh，并仅保留 DMG/ZIP 手动安装。 |
| WebSocket subprotocol 与旧客户端不兼容 | 旧 query token 客户端仍可用但不推荐 | 服务端暂兼容 header/query/subprotocol，客户端改为 subprotocol | 回退客户端构造 URL。 |
| CSP 过严影响开发服务器 | Renderer 开发热更新或本地 API 连接失败 | 允许 `127.0.0.1`、`localhost` 的 http/ws | 临时放宽 `connect-src`，不放宽 object/frame。 |
