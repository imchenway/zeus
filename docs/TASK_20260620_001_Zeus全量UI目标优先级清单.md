## 1. 设计总原则

### 1.1 信息架构原则

- 外层只有一个项目优先 source-list。
- 项目展开后只有 `任务 / 代码 / 会话` 三个项目内入口。
- 项目设置通过项目行的小设置按钮进入。
- 全局设置通过左下角固定设置按钮进入。
- 不允许任务页、代码页再长出第二个左侧导航。
- 不允许用 CSS 隐藏旧 DOM 冒充结构完成，必须从渲染结构中移除旧边栏。

### 1.2 视觉原则

- 采用 macOS / Apple 风格：轻底色、细分隔、低饱和 hover、克制圆角、系统字体、自然密度。
- 禁止卡片堆叠、后台面板、重复大标题、松散胶囊按钮、老年机字号、过黄或过青的底色。
- 页面只展示当前任务路径需要的信息，低频操作进入更多菜单、抽屉或二级面板。
- 视觉参考 Codex 的 source-list、新建对话、会话输入框和 Apple Support Communities 的极简层级与留白。

### 1.3 交互原则

- 高频操作露出，低频操作收纳。
- hover 只提示当前可操作区域，鼠标移开后不保留多余按钮。
- selected 只标识当前真正选择项，不允许 A 项目任务选中导致 B 项目任务也灰底。
- 项目展开箭头必须有交互，展开后朝下，收起后朝右。
- 多个项目允许同时展开。
- 动效只表达状态变化，不做装饰性表演。
