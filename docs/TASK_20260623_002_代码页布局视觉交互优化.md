# TASK_20260623_002 代码页布局视觉交互优化

## 1. 需求与目标

用户选择先优化 Zeus「代码页」。本轮不是重做代码页信息架构，而是在既有“代码图谱是主角”的契约上，针对真实截图和源码审查暴露的布局、视觉、交互问题做一次符合规范的收口。

唯一核心目标：进入代码页后，用户优先理解当前项目的真实代码图谱，并能直接从图谱节点继续打开源码或创建任务。

## 2. 当前问题

1. **小型真实图谱空白过大**：当服务端返回 3 到 8 个节点的小图谱且坐标来自较大的服务端布局时，前端照搬坐标会把节点分散到画布边缘，主舞台出现大面积无意义空白，削弱“图谱是主角”的视觉焦点。
2. **键盘交互语义不完整**：普通图谱节点和时序图 lifeline 已支持 Enter/Space/O/T 等键盘动作，但 DOM 没有声明 `aria-keyshortcuts`，屏幕阅读器和键盘用户不容易发现快捷键。
3. **风险边界**：不能破坏已有大图谱压缩、项目图谱隔离、时序图语义、Runtime 辅助带和二级工具收纳契约。

## 4. 契约变更

### 4.1 UI / 交互契约

- 代码页仍保持：图谱主舞台在首层，仓库上下文和操作 rail 退到辅助层。
- 3 到 8 个节点的小型服务端图谱会在前端做等比例收束：保留节点相对结构，但避免把节点贴到画布边缘或形成大空白。
- 1 到 2 个节点的服务端布局继续尊重原始坐标，避免破坏精确布局测试和已有双节点图谱表达。
- 超大服务端布局仍走既有坐标压缩，不允许撑爆 macOS 窗口。
- 图谱节点和时序 lifeline 明确暴露：`Enter/Space` 选中，`O` 打开源码，`T` 创建任务。
- 图谱边、时序消息、分支 fragment、内联源码/任务动作明确暴露 `Enter/Space`。

### 4.2 API / 数据契约

- 不新增后端 API。
- 不新增数据库表或字段。
- 不改变 `GraphViewSnapshot` wire shape。
- `buildGraphCanvasLayout` 从 renderer 内部函数改为可测试导出，属于前端内部测试接口，不对后端或外部插件承诺稳定。

## 6. 实施顺序

1. 读取 `DESIGN.md`、`PRODUCT.md` 与代码页现有测试，确认代码页核心目标仍是图谱主舞台。
2. 增加红灯测试：小型服务端图谱必须收束在画布中心有效区域；图谱节点/边必须暴露键盘快捷键语义。
3. 修改 `buildGraphCanvasLayout`：仅对 3 到 8 个节点且来自服务端布局的小图谱做等比例收束。
4. 为普通图谱、时序图和内联动作补 `aria-keyshortcuts`。
5. 运行聚焦测试与代码页全量测试。
6. 执行 lint、typecheck、format、diff 检查；需要交付时再跑 `pnpm package:mac`。

## 7. 风险与回滚

| 风险 | 控制 |
| --- | --- |
| 小图谱收束影响服务端精确坐标 | 只作用于 3 到 8 个节点；1 到 2 个节点保留原坐标；超大图谱仍由既有压缩逻辑保护。 |
| 键盘快捷键声明与实际行为不一致 | 测试同时覆盖 DOM 语义和现有 keydown handler；快捷键不新增新行为，只公开已有行为。 |
| 导出内部布局函数增加误用风险 | 仅供 renderer 测试导入，不进入 API client 或后端契约。 |
| 视觉仍需真实窗口截图复核 | 本轮先完成源码与渲染契约；若要宣称响应式视觉完成，需要刷新截图矩阵并执行 package gate。 |

回滚方式：移除新增测试、撤回 `buildGraphCanvasLayout` 导出与 `compactSmallGraphCanvasLayout` 调用、删除新增 `aria-keyshortcuts` 属性即可；不涉及数据迁移。
