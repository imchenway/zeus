# TASK_20260623_003 会话页布局视觉交互优化

## 1. 背景与目标

用户要求基于截图找出某个页面的布局、视觉、交互问题并优化。本轮最新工具选择为“会话页”。

审查截图：

- `docs/artifacts/TASK_20260620_001_screenshots/png/conversations-360.png`
- `docs/artifacts/TASK_20260620_001_screenshots/png/conversations-480.png`
- `docs/artifacts/TASK_20260620_001_screenshots/png/conversations-768.png`
- `docs/artifacts/TASK_20260620_001_screenshots/png/conversations-1024.png`
- `docs/artifacts/TASK_20260620_001_screenshots/png/conversations-1440.png`

目标：让会话页在 360/480/768 紧凑宽度下仍以“当前线程 + composer”为核心，不让全局 source-list、会话列表、空态和大输入盒抢占首屏。

## 2. 截图问题

| 编号 | 现象 | 影响 | 根因判断 |
|---|---|---|---|
| P1 | 360/480 下全局 source-list 占用接近半屏 | 用户进入会话页后要先滚动才能看到对话主任务 | 极窄壳层只把侧栏转到顶部，但最大高度仍偏大 |
| P2 | 768 下会话列表空态占据过多垂直空间 | 当前线程与输入区被推到下方，composer 首屏可见性差 | 会话页双栏收窄后列表仍保持桌面高度 |
| P3 | 768 下 composer 内输入内容出现竖排/窄列观感 | 输入区不可信，像被挤坏的表单 | 紧凑宽度下 composer padding、textarea 高度和工具栏密度未单独收敛 |
| P4 | 360/480 下 composer 需要滚动较多才到达 | 高频发送动作不在浅层 | 消息区、列表区和 composer 间距仍按桌面节奏 |

## 3. 页面核心目标判断

1. 唯一核心目标：选择或查看当前会话，并在底部 composer 继续推进任务。
2. 用户当前最需要做什么：阅读当前任务状态、输入补充要求、发送到 CLI 或打开上下文抽屉。
3. 首层必须展示：当前线程标题、最新消息/等待状态、composer 输入与发送动作。
4. 应收纳到下一层：运行环境、上下文、代码变更、模板、完整 Runtime 日志。
5. 禁止回退：不能把会话页做成任务管理页、卡片墙或后台表单；只有会话页允许左侧会话列表 + 右侧详情。

## 6. 契约变更

### 6.1 CSS 布局契约

- `@media (max-width: 860px)`：
  - `.workspace-view-project-sessions > .conversation-list-pane` 最大高度收敛为 `min(24vh, 220px)`。
  - `.conversation-message-stage` 使用更小间距和 `padding: 6px 12px 0`。
  - `.conversation-codex-composer` 宽度改为 `min(100%, 680px)`，降低阴影与圆角体量。
  - `.conversation-codex-composer .conversation-compose-field textarea` 和已有对话 textarea 最小高度降为 `56px`。
  - 新建对话 textarea 最小高度降为 `72px`。
- `@media (max-width: 560px)`：
  - 全局 project source-list 最大高度收敛为 `28vh`，工作区最小高度提升为 `72vh`。
  - 空会话行、composer 工具栏和模型 chip 进一步压缩，避免首屏只剩导航和空态。

### 6.2 交互契约

- 会话页紧凑宽度下仍保留会话列表，但降低其首屏占比。
- Composer 仍在当前页面内，不转成弹窗，不隐藏发送按钮。
- Runtime、上下文、代码变更和模板继续通过既有抽屉进入，不在首屏平铺。

6. 运行 lint/typecheck/format/diff 检查与 macOS 打包门禁。

## 9. 风险与回滚

| 风险 | 影响 | 控制 | 回滚 |
|---|---|---|---|
| 会话列表在窄屏下变矮 | 用户需要滚动左侧列表查看更多会话 | 会话列表仍可滚动，且只在 `<=860px` 生效 | 删除 `会话页截图驱动紧凑布局最终覆盖` CSS 块和对应测试。 |
| Composer 高度变低 | 长输入需要 textarea 内滚动或继续扩展 | 只降低最小高度，不限制用户输入值 | 恢复 textarea `min-block-size` 到原值。 |
| 极窄 source-list 高度变小 | 项目列表需要滚动 | 只在 `<=560px`，目的是保证工作区可见 | 删除 `max-block-size: 28vh` 和 `min-block-size: 72vh` 覆盖。 |
