# TASK_20260623_004 无项目侧栏布局视觉优化

## 1. 背景与目标

用户要求通过截图任意找出一个页面的布局、视觉或交互问题并优化。本轮选择 Zeus 启动后的「无项目 / 新对话」页面，因为这是新用户第一次进入 Zeus 的关键路径。

截图来源：

- 修复前：`/tmp/zeus-home-1440.png`
- 修复后：`/tmp/zeus-no-project-after-1440.png`

唯一核心目标：当本地服务未连接且没有项目时，左侧 source-list 的首个真实任务必须清楚告诉用户“选择真实本地代码库”，不能被右侧状态按钮挤压成省略号。

## 2. 截图问题

| 编号 | 现象 | 影响 | 根因判断 |
|---|---|---|---|
| P1 | 左侧项目空态中，“选择真实本地代码库”被挤成“选择真...” | 新用户无法快速识别第一步是选择本地仓库，Aha Moment 前置动作不清楚 | `.project-inline-recovery-row` 使用 `minmax(0, 1fr) auto` 双列布局，右侧状态/按钮占用宽度 |
| P2 | 状态按钮像 badge 一样贴在标题右侧 | 视觉上主任务和失败状态争抢优先级 | 侧栏窄列中仍沿用通用 command rail 横向布局 |
| P3 | 状态/按钮可点击目标区域偏小 | 本地服务恢复后，选择仓库入口不够像一个明确操作 | command rail 未在侧栏空态下拉伸到整行 |

## 3. 页面核心目标判断

1. 唯一核心目标：引导用户选择真实本地代码库，进入项目扫描与代码图谱主路径。
2. 用户当前最需要做什么：理解当前没有项目，并等待/触发本地仓库选择。
3. 首层必须展示：项目空态标题、当前本地服务状态/选择仓库按钮。
4. 应收纳到下一层：项目配置、扫描设置、运行环境、Git 详情。
5. 禁止回退：不能把左侧栏做成大卡片墙，也不能把项目选择动作塞回页面大标题或独立 Dashboard。

## 4. 受影响目录

| 路径 | 影响 |
|---|---|
| `apps/desktop/src/renderer/styles.css` | 增加 no-project source-list recovery 的侧栏专用覆盖：单列排布、标题正常换行、按钮整行拉伸。 |
| `apps/desktop/test/app-shell-layout.test.tsx` | 新增截图驱动 CSS 契约测试，锁定侧栏空项目态不能再被按钮挤压标题。 |
| `docs/TASK_20260623_004_无项目侧栏布局视觉优化.md` | 记录截图问题、契约变更、验证和风险。 |

## 5. 不受影响边界

| 范围 | 是否修改 | 原因 |
|---|---:|---|
| 后端 / SQLite / IPC | 否 | 本轮只改 renderer CSS 与测试，不改项目创建数据流。 |
| Electron 本地桥接 | 否 | 截图中的本地服务失败是浏览器预览环境事实，不用假状态掩盖。 |
| Runtime / Git / Telegram | 否 | 无项目首屏引导不改变这些模块。 |
| 构建依赖 / CI | 否 | 不新增依赖，不改构建配置。 |

## 6. 契约变更

### 6.1 UI / 布局契约

- `.macos-ai-app.zeus-shell .project-first-sidebar .project-inline-recovery-row` 在侧栏内改为单列布局。
- `.project-inline-recovery-copy > strong` 允许正常换行，不再强制 `white-space: nowrap`。
- `.project-inline-recovery-command-rail` 在侧栏内左对齐。
- `.project-inline-recovery-command-rail button` 在侧栏内占满可用宽度，提升可读性和可点击区域。

### 6.2 API / 数据契约

- 不新增 API。
- 不新增数据库表或字段。
- 不改变 `DashboardSnapshot`、项目创建、仓库选择或本地服务状态语义。
