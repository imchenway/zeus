Zeus 每次启动弹出钥匙串密码：故障排查图

主结论：弹窗来自 Electron/Chromium Safe Storage；当前重复提示最可能与“只点允许”或未正式签名/ad-hoc 构建身份不稳定有关。

证据阶梯

  • E1截图 item：`@zeus/desktop Safe Storage`，符合 Electron/Chromium Safe Storage 命名。
  • E2`packages/security-core/src/index.ts` 自定义 Keychain service 默认是 `Zeus`，与截图不同。
  • E3`apps/desktop/package.json` 包名是 `@zeus/desktop`。
  • E4运行进程使用 `~/Library/Application Support/@zeus/desktop` 作为 user data。
  • E5`apps/desktop/electron-builder.yml` 配置 `identity: null`。
  • E6当前 `Zeus.app` 为 `Signature=adhoc`、`TeamIdentifier=not set`。
  • E7login keychain 当前 `no-timeout`，暂不支持“定时锁定导致每次问”的主因判断。

假设裁决

  • 成立截图来源是 Electron/Chromium Safe Storage，而不是 Zeus 自定义 SecretStore。
  • 高疑未正式签名/ad-hoc 构建和频繁重新打包导致 Keychain 反复授权。
  • 高疑如果每次点的是“允许”，macOS 只授权本次访问。
  • 待验若点过“始终允许”仍在同一个未变 app 上复现，需要检查 Safe Storage item ACL 是否异常。
A. 临时处理

同一 App bundle 上点“始终允许”。最快；重新打包后可能再问。

B. 清理旧 item

只删除 `@zeus/desktop Safe Storage` 后重建。注意不要删服务名 `Zeus` 的业务密钥。

C. 稳定命名/profile

把 app name/userData 收敛为 `Zeus`，需要迁移和一次新授权。

D. 正式签名/公证 🌟

Developer ID + notarization 提供稳定身份，是长期日用和发布正解。
