# TASK_20260627_002_Zeus左侧栏顶部留白压缩

## 问题

用户连续两次截图指出左侧栏红黄绿按钮下方到「新对话 / 搜索」之间仍存在大块空白，并追问“这些留着干嘛？”。结论：这块空间没有业务价值。第一次从 104px 压到 72px 后，按 Retina 物理像素看仍接近一整块空白，因此本轮继续把 hidden titlebar 顶部避让压缩到 44px。

## 设计判断

- 唯一核心目标：左侧 source-list 首屏直接服务「新对话 / 搜索 / 项目选择」。
- 用户当前最需要做：快速开始新对话或搜索项目，不应先扫过无意义空白。
- 首层必须展示：红绿灯安全区下方立即展示快捷入口。
- 应收纳/避免：不保留占位 spacer，不把避让责任交给内容流，不新增标题或说明。
- 卡片/标题/功能平铺检查：本轮不新增任何卡片、标题或面板，只压缩 source-list 顶部 inset。

## 实现结论

| 项 | 调整前 | 首次修正 | 本轮最终 |
| --- | --- | --- | --- |
| 顶部安全区 token | `--zeus-hidden-titlebar-safe-top: 104px` | `72px` | `44px` |
| 红绿灯不可读内容区 | `--zeus-traffic-light-no-content-zone: 72px` | `72px` | `44px` |
| DOM inline style | 104px fallback | 72px fallback | 44px fallback |
| 占位 spacer | 已 collapsed | 继续 collapsed | 继续 collapsed |

44px 是当前更符合截图反馈的边界：仍保留 macOS 左上窗口按钮的垂直避让，但不再为红绿灯单独保留一整行空白。后续如果真实设备出现按钮遮挡，应按“最小回升”调到 52/58px，而不是回到 72/104px。

## 契约变更

- UI 契约：左侧栏顶部只保留 44px 红绿灯必要安全区，不保留额外空白。
- 可访问性契约：`project-window-control-reserved-space` 仍 `aria-hidden` 且 `display:none`；source-list 键盘与 ARIA 语义不变。
- 数据/API/数据库：不变。
- 构建/依赖：不新增依赖，不改构建配置。

## 实施顺序

1. 先把布局测试预期从 72px 改为 44px，并确认现实现红灯失败。
2. 同步 `App.tsx` inline style 与 `styles.css` 顶部安全区 token。
3. 运行聚焦测试、壳层全量测试、类型检查和旧值反向扫描。
4. 退出运行中的 Zeus，再执行 `pnpm package:mac` 更新打包产物。

## 风险与回滚

- 风险：如果某些 macOS / Electron 版本红绿灯垂直位置比当前更低，44px 可能接近快捷入口。
- 监控口径：以真实打包 App 截图为准；若出现遮挡，只做 52/58px 的最小回升。
- 回滚：把 `--zeus-hidden-titlebar-safe-top`、`--zeus-traffic-light-no-content-zone` 与 App inline fallback 恢复到 72px，并恢复测试断言；不推荐回到 104px，因为会复现用户截图中的死留白。
