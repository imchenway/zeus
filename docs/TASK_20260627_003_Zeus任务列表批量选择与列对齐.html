Zeus 任务列表：批量选择与列对齐设计

目标：把任务页从“整行按钮列表”收敛成可选择、可批量管理、列轨道稳定的数据表工作台。

现象无批量选择；列头和字段视觉不齐。
根因任务行是整行 button，无法安全容纳 checkbox；列轨道分散到独立 grid。
推荐改为可选择表格行 + 批量操作栏，复用现有单任务 API。
边界不改 DB，不新增依赖；批量删除不进首轮默认入口。

批量动作矩阵

动作适用状态处理
状态修改按状态机逐项判断目标状态下拉，合法项调用 updateTaskStatus 或专用运行控制 API
删除默认排除 running / waiting_confirmation二次确认后调用 deleteTask，失败项保留选择

实施顺序

  1. 1先补红灯测试：选择模型、bulk eligibility、列轨道统一。
  2. 2改 TaskWorkspace 行结构：checkbox 独立，行内容打开详情。
  3. 3App 层保存 selectedTaskIds，批量动作限并发执行。
  4. 4CSS 最终覆盖：选择列固定，header/row 同轨道。
  5. 5跑任务页回归、typecheck、打包门禁。
D1 首轮批量动作范围：已更新为 AA = 批量状态修改 + 批量删除。删除必须二次确认。
D2 实现层级：推荐 AA = 前端批量编排复用现有 API。无需改 DB/API，风险更低。
D3 选择范围：推荐 AA = 只选择当前筛选结果，不跨项目、不跨归档列表。
D4 批量删除边界：推荐 AA = 默认跳过 running / waiting_confirmation。若需要强删所有状态，需输入确认短语。
