# TASK_20260627_004_Zeus项目设置抽屉与空任务态视觉返修

## 需求来源

用户连续反馈两处界面质量问题：

1. 项目设置抽屉「丑、潦草」，在普通桌面分辨率下不能自适应，表单控件被撑开并出现裁切。
2. 任务页空状态只显示一条粗糙的「还没有任务」横幅，没有说明下一步，也不符合 Zeus 的产品 UI 质量底线。

## 现象 -> 影响 -> 根因 -> 修法 -> 验证

| 项 | 项目设置抽屉 | 任务空状态 |
| --- | --- | --- |
| 现象 | 右侧抽屉继承 980px 宽抽屉，顶部标题/关闭按钮像调试面板，配置表单在普通窗口下仍按桌面两列铺开。 | 空列表区域是一条满宽灰底横幅，只显示标题。 |
| 影响 | 控件在 13/14 寸 MacBook 或默认窗口宽度下容易裁切，用户无法稳定编辑配置。 | 用户不知道这里会出现什么、为什么为空、下一步怎么做；视觉像占位条。 |
| 根因 | 抽屉宽度和断点按 viewport 写死；项目配置行又被全局输入行覆盖为两列后台表单。 | 复用了通用 inline recovery row，缺少任务页专属 empty state 信息架构。 |
| 修法 | 给 `.project-drawer` 加容器查询、窄幅 sheet 宽度、隐藏重复 chrome 文案、控件列最大宽度和 680px 容器内单列回退。 | 增加 `emptyHelp` 文案、`task-empty-state` 结构和专属样式：收窄内容、解释下一步、不重复堆按钮。 |
| 验证 | 新增测试锁定项目抽屉宽度、container query、隐藏 chrome、控件 100% 收缩。 | 新增测试锁定空态 class、说明文案、收窄样式和可读排版。 |

## 契约变更

### 前端 UI 契约

- 项目设置抽屉：
  - 最大宽度收敛为 `760px`，不再继承全局 `980px` 抽屉。
  - 以抽屉容器宽度为断点，`680px` 以下表单切到单列。
  - 顶部抽屉 chrome 只保留右上关闭入口，重复标题做视觉隐藏但保留 `aria-label`。
  - 输入、下拉、文本域必须 `inline-size: 100%` 且不撑破容器。
- 任务空状态：
  - 空任务时显示标题 + 下一步说明。
  - 不在空态内重复「新任务」按钮，主操作仍保持在 toolbar。
  - 空态内容收窄，不再显示满宽粗糙横幅。

### API / 数据库 / 后端契约

- 不改 API。
- 不改数据库表、字段、索引。
- 不新增依赖。
- 不改构建或 CI。

## UI/UX 判断

### 页面唯一核心目标

- 项目设置抽屉：编辑当前项目配置。
- 任务页空状态：告诉用户当前任务列表为空，并指向第一步创建任务。

### 用户当前最需要做

- 抽屉：在当前窗口宽度下稳定阅读和编辑配置。
- 空态：理解为什么没有任务，并知道从 toolbar 的「新任务」开始。

### 首层必须展示

- 抽屉：当前配置摘要、常用配置项、保存动作。
- 空态：空状态标题、真实用途说明、下一步提示。

### 应收纳到下一层

- 抽屉：高级配置仍在抽屉内部滚动，不铺到主页面。
- 空态：不把运行、详情、证据等低频内容塞进空态。

### 避免项

- 不恢复后台表单式满宽布局。
- 不用卡片堆叠或大 HERO。
- 不让空态成为重复按钮区。
