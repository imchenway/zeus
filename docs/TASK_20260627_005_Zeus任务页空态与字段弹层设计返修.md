# TASK_20260627_005_Zeus任务页空态与字段弹层设计返修

## 结论

用户截图指出的问题成立：任务页空态和字段弹层仍停留在工程占位层，没有达到 Zeus `DESIGN.md` 要求的“空状态说明下一步”和“popover 可用、可读、可退出”。本轮按缺陷返修处理，目标是把两个点从“能显示”提升为“能作为产品界面使用”。

## 用户反馈与现象

- 空任务页：宽屏下只有一条很小的文字提示，周围是大片无意义空白，缺少主操作和任务运行后会出现什么的可视化预期。
- 字段弹层：从 `…` 打开后像一块粗糙白板，字段过多时被窗口裁切，缺少标题、说明、滚动边界和底部动作区。

## 影响

- 新用户不知道下一步应该创建任务，也不知道任务列表将承载状态、AI 执行和证据。
- 字段管理在小窗口和普通 MacBook 分辨率下不可用，用户看不到完整字段，也难以恢复默认字段。
- 视觉层级与 Zeus 生产力工具定位不一致，容易退回“后台面板 / 工程占位”体验。

## 根因

1. 空态只把 `emptyTitle + emptyHelp` 放到一个轻量行里，没有承担任务页首屏引导职责。
2. 字段设置浮层没有固定头尾和独立滚动列表，所有字段直接铺在一个绝对定位容器里。
3. 之前测试只校验 DOM 存在和基础布局，没有锁定“弹层边界”和“空态 starter surface”。

## 修法

### 视觉与交互 thesis

- visual thesis：任务页空态必须像任务工作台的起点，而不是空白画布上的一句话。
- content plan：空态首层只保留“还没有任务 / 说明 / 新任务 / 三个结果预览”，不把设置、历史或详情塞进首屏。
- interaction thesis：主动作直接创建新任务；字段弹层固定 header、滚动 list、footer，Escape / 外部点击关闭逻辑沿用原有契约。

## 风险与回滚

- 风险：空态新增第二个“新任务”入口，可能与顶部按钮形成重复。当前保留原因是空态本身就是任务创建主路径，且低频字段入口仍在顶部 `…`。
