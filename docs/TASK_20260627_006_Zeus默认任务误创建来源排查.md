# TASK_20260627_006_Zeus默认任务误创建来源排查

## 结论

截图中的 `ZEU-000001 / 分析当前项目结构` 是 Zeus 前端“默认任务草稿”动作创建出来的任务，不是项目扫描自动生成，也不是数据库启动种子。

本轮已修复：会话页的“新对话 / 空列表新对话 / 新对话发送 / 完成一次项目任务”不再调用任务创建链路；只有任务页明确点击“新任务”才会创建正式 `tasks` 记录。

## 修复前可触发入口

这些入口曾触发同一个默认任务创建函数：

- 任务页顶部 `新任务`。
- 会话页左栏 `新对话`。
- 空会话列表里的 `新对话`。
- 新对话输入框的发送按钮。
- 新对话建议里的 `完成一次项目任务`。

其中“会话页新对话 / 发送”触发“任务创建”是语义错误：用户以为在开对话，结果任务列表里多出 `分析当前项目结构`。

## 根因

会话页复用了任务系统作为会话承载，但 UI 文案叫“新对话”，实现却绑定到 `createDefaultTask()`。这导致非任务入口也会生成任务记录。

## 本轮修法

### 受影响目录

- `/Users/david/hypha/zeus/apps/desktop/src/renderer/App.tsx`
- `/Users/david/hypha/zeus/apps/desktop/test/app-shell-layout.test.tsx`
- `/Users/david/hypha/zeus/docs/TASK_20260627_006_Zeus默认任务误创建来源排查.md`
- `/Users/david/hypha/zeus/docs/TASK_20260627_006_Zeus默认任务误创建来源排查.html`

### 契约变更

- 会话入口只打开本地会话草稿，不写 `tasks` 表，不生成 `ZEU-xxxxx` 编号。
- 新对话草稿用 `conversationDraftOpen` 表达，避免自动回退到第一条任务线程。
- 任务页 `新任务` 继续绑定 `createTaskFromTaskToolbar()`，是正式任务创建入口。
- 新对话建议里的“完成一次项目任务”改为跳转任务页，由用户再明确点击“新任务”。
- 从图谱节点、任务模板等明确任务动作创建任务时，会关闭会话草稿状态。

### 关键实现

- 删除会话入口对 `createDefaultTask()` 的绑定。
- 新增 `prepareNewConversationDraft()`，只做前端草稿态、筛选清理和导航，不调用 `onCreateDefaultTask`。
- 保留 `createProjectDefaultTask({ openCreatedTaskDrawer: true })` 给任务页“新任务”。
- 任务列表选择真实任务时关闭会话草稿，恢复已有会话线程。

## 风险与回滚

- 风险：会话体系目前仍未独立成真正 conversation 数据模型；本轮只阻止误写任务表，不实现新的会话持久化。
- 风险：新对话输入框当前仍是前端草稿态，发送不会生成官方任务；后续若要支持独立会话，需要新增 conversation API，而不是复用 tasks。
- 回滚：恢复会话入口到 `createDefaultTask()` 即可回到旧行为，但会重新产生误创建任务问题，不推荐。

## 已有误创建任务处理

本轮未直接删除用户本机任务数据。已经误创建的 `ZEU-000001` 应走正式 UI 删除或批量删除链路，避免绕过事件与审计记录。
