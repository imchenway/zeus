说明

现场五次返修：任务说明粘贴图片无反应

根因：旧逻辑只读浏览器文件项和 Electron bitmap，未覆盖 macOS/Finder/Paste.app 常见的 file-url、uri-list、HTML file src/href、本地路径文本，以及 typed-array IPC 载荷。

修法：新增原生剪贴板附件解析，创建任务表单级 paste 处理；普通文字不拦截，图片/文件引用转为本机附件路径。

现场六次返修：粘贴入口不再猜测，补齐 Paste.app 私有格式

根因：renderer 仍在猜 clipboardData 是否像附件；Paste.app/macOS 可能只把图片文件路径藏在原生私有格式里，浏览器 paste 事件看起来像普通文本。

修法：创建任务文本控件粘贴时先读原生附件；读不到附件再按原光标位置插回普通文字。Main 进程补齐 native image buffer、HTML data:image、未知小体积私有格式、嵌入 file:// 与 /Users/... 路径解析。

验证：新增私有剪贴板红绿测试；聚焦回归 6 passed；typecheck/lint/format/diff check 通过；全量测试 66 files / 987 tests 通过；package:mac 与产物 codesign 校验通过。

现场七次返修：原生剪贴板改为 Main 进程直接保存

现场证据：当前 macOS 剪贴板是 public.png；Electron 可读到 image/png，readImage 非空，并能解析出 PNG 附件。

根因判断：图片读取已成立，风险在旧链路让二进制在 Main → renderer → Main 往返，失败时界面没有附件反馈。

修法：新增 zeus:save-task-clipboard-attachments，由 Main 进程直接读取原生剪贴板并保存到 userData/task-attachments，renderer 只接收本机路径并合并附件列表。

验证：聚焦回归 6 passed；静态门禁通过；全量测试 66 files / 987 tests 通过；package:mac、产物检查与 codesign 通过。

现场八次返修：本地图片文件粘贴没有 DOM paste 事件

现场证据：在打包产物中聚焦 #task-create-description-input，捕获级 paste 监听记录仍为 pasteEvents: []；旧逻辑只在 onPaste 中工作，因此本地图片文件按 ⌘V 时没有任何触发点。

修法：创建任务弹窗新增 Cmd/Ctrl+V 键盘兜底，不阻止默认粘贴；如果短暂等待后没有 DOM paste 事件，就直接调用 Main 进程 saveTaskClipboardAttachments() 保存原生剪贴板图片/文件。

真实电脑验证：复制本地 PNG 文件（剪贴板类型 «class furl»）后向真实 Zeus 窗口发送 ⌘V；DOM 仍无 paste 事件，但附件列表出现 pasted-task-screenshot-*.png，保存路径位于本机 Application Support/@zeus/desktop/task-attachments，无错误提示。

验证：TDD 红灯命中缺少 pasteShortcutFallbackTokenRef；修复后聚焦回归 6 passed，typecheck/lint/format/diff check 通过，全量测试 66 files / 987 tests 通过，package:mac 通过。
