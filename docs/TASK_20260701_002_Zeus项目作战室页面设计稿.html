页面设计稿：Zeus 项目作战室，把代码图谱变成下一步行动

这不是给旧截图换皮，而是把 Zeus 的真实产品边界重新组织成一个“先理解项目，再进入任务”的工作台：左侧是项目源，中央是可追溯代码图谱，右侧是选中对象的证据与动作，底部把理解结果接到任务、运行和 Git Diff。

不发散候选的理由

用户当前要求“画出你的设计来”，前一轮已收敛到“项目作战室”方向；本轮输出一个可评审的最终稿，而不是 A/B/C 换色稿。已明确剔除过期截图，依据只来自当前仓库文档与代码锚点。

Visual thesis克制、冷静、低疲劳的 macOS 工具界面；白瓷底、轻网格、少量蓝绿状态色，不做营销式高饱和。
Content plan首屏只服务一条主路径：选择真实项目 → 阅读代码图谱 → 选中节点 → 创建任务或打开源码。
Interaction thesis图谱节点点击驱动右侧 inspector；任务动作从当前对象生成；搜索、运行、Diff 作为上下文动作而非独立页面噪音。
Accessibility thesis所有主操作可键盘触达；焦点外显；状态不只靠颜色；reduced-motion 下关闭漂移动效和 shimmer。

One-hop relationships

  • 节点点击驱动 inspector；键盘可浏览节点；图谱不再被卡片列表稀释。
  • 与 TaskWorkspace、Runtime、Git Diff 形成任务闭环。

Collapsed details

  • 日志、SQL、长命令、边详情放入可展开区；首屏只保留判断所需事实。
  • 危险操作进入二次确认；不可用动作展示原因，而不是隐藏。

主要状态在首层可见

Zeus 的产品原则要求没有真实数据就展示空、失败、未配置或等待状态；因此设计稿把这些状态作为系统部件，而不是事后补丁。

Loading局部骨架保留已有真实数据，不整页遮罩。
Empty说明这里会出现什么、为什么重要、下一步怎么做。
Error显示原因、恢复动作和可复制诊断信息。
Permission denied说明本地权限边界与授权路径。
External wait等待 CLI、Telegram 或本地服务时显示来源与下一步。

响应式口径

桌面是主生产力场景；窄屏不把整张图缩成不可读缩略图，而是改为“节点列表 + inspector 抽屉 + 任务闭环”的纵向节奏。

Desktop ≥ 1180px

左源列表常驻，中间图谱主舞台，右侧 inspector 常驻；适合长时间研发。

Tablet 760–1179px

inspector 下沉为横向分区，避免主画布过窄；source-list 可缩短。

Mobile / narrow 390px

图谱转纵向节点链路；详情抽屉在节点之后出现；不依赖横向拖动理解页面。

收纳与不平铺

  • 首层只展示当前项目、当前对象、当前状态和主操作。
  • 日志、长证据、二级工具和危险动作进入 inspector 或折叠区。
  • 不使用重复同款卡片网格，不把任务、设置、会话信息架构混在一屏。

可访问性与状态

  • 节点、按钮、导航全部可键盘聚焦，并有可见焦点环。
  • aria-current、aria-pressed、状态文本与颜色同时表达选择和异常。
  • reduced-motion 下禁用节点漂移和 loading 动效；正文字号不小于 12px。
