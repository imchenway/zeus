# TASK_20260701_003 Zeus 最新页面设计重构建议

## 目标

基于用户本轮提供的最新 Zeus 页面截图，重新评估当前产品 UI，并给出设计优化与重构建议。上一轮旧截图已排除，不作为本轮设计依据。

## 本轮结论

当前 Zeus 最新页面不是“视觉太差”，而是“页面都已成型，但主路径分散”：任务、代码、会话、设置分别有自己的操作区域、详情层和空态表达，尚未统一成“当前项目 / 当前对象 / 当前状态 / 下一步行动”的项目工作台语法。

推荐方案：保留当前 macOS source-list、克制中性色与真实状态原则，优先重构为三个共享 Shell：

1. Project Workspace Shell：用于 Code Map、Sessions、Runtime、Git Diff。
2. Work Queue Shell：用于 Tasks 与任务详情。
3. Configuration Shell：用于 Settings、项目设置与安全配置。

## 优先级建议

1. P0：统一 object-toolbar 与 context-inspector，让所有项目内页面稳定展示当前对象、状态和主操作。
2. P1：把 Code Map 作为项目作战室主舞台，强化“从图谱节点创建任务 / 打开源码 / 问图谱”。
3. P1：把 Tasks + Sessions 合成任务执行线索，减少全屏抽屉和孤立 composer。
4. P2：设置页改为状态驱动配置工作台，突出 AI CLI、Telegram、安全、发布等待项。
