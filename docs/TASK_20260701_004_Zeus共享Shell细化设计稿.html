细化设计稿 · 共享 Shell · 不改源码

页面设计稿：Zeus 共享 Shell 细化稿，三个页面共用同一套“对象与行动”语法

这版把上一轮建议落到可评审的页面结构：Code Map 成为项目作战室主舞台，Tasks 和 Sessions 合成执行线，Settings 变成状态驱动配置工作台。设计保持当前 Zeus 的克制 macOS 工具感，不做换皮。

Brief 已锁定

产品：Zeus 本地优先 macOS AI 研发工作台。视觉来源：用户提供的最新截图、PRODUCT.md、DESIGN.md。交互级别：静态细化稿，带关键状态与响应式口径，暂不进入源码实现。

visual thesis浅色优先、冷中性、低疲劳；像可靠的本机开发工具，而不是营销型 SaaS。
content plan首屏固定四件事：当前项目、当前对象、状态、下一步。长证据进入 inspector。
interaction thesis选择图谱节点、任务或设置组时，只刷新右侧上下文；主舞台不跳页。
accessibility thesis所有 icon-only 操作都有名称，焦点顺序固定，状态有文本，reduced-motion 下无装饰动效。
不发散候选：用户要求继续细化已认可的方向，本稿不提供 A/B/C 视觉方向，而是把推荐方向拆成三张高保真页面稿与组件规则。

组件细化规则

后续如果进入开发，先把这些规则落到组件契约，再逐页替换，不要用 CSS 隐藏旧结构冒充完成。

object-toolbar

每页第一行稳定回答：当前项目、当前对象、真实状态、主操作。

  • 高度 58px，可紧凑双行。
  • 低频操作进更多菜单。
  • 状态用文本 + 色点，不只靠颜色。

context-inspector

承载选中节点、任务、配置组的证据与二级动作。

  • 桌面右侧 318px。
  • 平板下沉，移动端在对象后方。
  • 长日志和来源清单折叠。

decision-rail

统一任务运行、Git 写操作、取消任务和保存反馈。

  • 危险动作必须说明影响。
  • 二次确认与审计记录不可省。
  • 失败保留用户输入。

composer

会话、任务续写、图谱问答共用底部输入语法。

  • 显示当前上下文。
  • 发送按钮必须说明去向。
  • 禁用态显示原因。

关键状态细化

所有页面都必须有这些状态，不以假数据填充界面。

loading局部骨架，不遮住已有真实数据。
empty说明这里会出现什么、为什么重要、下一步怎么做。
error说明影响范围和恢复方式，不暴露敏感堆栈。
external waitAI CLI、Telegram、签名凭据等待用户配置。
saved / ready保存、扫描完成、运行就绪都显示更新时间和来源。

响应式细化

桌面是主场景，窄屏不把整张图缩成缩略图，而是改成纵向对象链路。

Desktop

source-list 常驻，stage 居中，inspector 常驻；适合外接屏和长时间研发。

Tablet

inspector 下沉或变二列，避免主舞台被压窄。

Narrow 390px

图谱节点转纵向列表，详情紧跟当前对象，不依赖横向滚动。

依据与边界

本稿是细化设计，不是实现验收；未修改源码。
