新的整体交互动线

Zeus 不应该让用户在任务、会话、代码、设置之间反复找入口。它应该把项目当工作容器，把任务当主对象，把 app-server / Runtime / CLI 对话当任务推进方式。

01全局命令入口⌘K 或左上新任务可以跨项目创建、搜索、继续、切状态。用户先表达“要推进什么”，再决定是否需要代码理解。
02项目任务 Lane首页不展示功能卡片，而展示各项目当前可行动任务。每条 Lane 都有下一步、Runtime 状态、证据摘要和主操作。
03任务内连续对话推送到 app-server 后，任务页就是对话页。继续、暂停、取消、标记完成都在同一个底部 composer 和状态 rail 内完成。
04代码理解降级为 Lens图谱和问答只在任务需要影响范围、依赖、表结构、方法逻辑时打开。输出回填到任务上下文，不独立成另一条工作线。
05证据优先收口每次执行沉淀任务事件、Runtime 日志、Git diff、附件和测试证据。完成不是按钮文案，而是证据链闭合。

高级精简样式规则

不要靠粗边框和多卡片建立结构。用材质层级、行密度、轻量阴影、hairline 分割和单一蓝紫强调色组织注意力。

Material珍珠白主工作区、雾灰 source-list、半透明工具条。避免纯黑纯白，也避免黑白块相交。
Chrome边框只作为 hairline 分隔，不给每块内容套完整框。低频信息用 drawer / lens 收纳。
Typography全系统字体，13px 主体、12px metadata、24px 工作区标题。靠字重和密度区分层级。
Motion120 到 200ms，只表达状态变化：任务进入运行、Runtime 等待、证据追加。reduced-motion 下取消 transform。
A11y项目导航 aria-current，任务 Lane role=list/listitem，状态 aria-live，按钮不小于 28px 高并有 focus-visible。
