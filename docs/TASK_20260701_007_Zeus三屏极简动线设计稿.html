返工结论 · 不做大一统工作台

页面设计稿：Zeus 三屏极简动线，每个页面只做一件事

你说得对，上一版越做越复杂，问题就是把项目、任务、会话、代码理解、证据全塞进一张“大驾驶舱”。这一版改成三层：入口只负责继续，项目页只负责任务队列，任务页只负责连续对话与状态推进。代码理解只在需要时进入，不常驻。

我对上一版的直接判断 Command Flow 的方向是对的，但呈现错了：它不应该是一个承载全部信息的页面，而应该是一条轻量路由。用户每一步只看到当前该做的事。
Rule 01一个页面，一个主动作

入口继续任务，项目页管理任务，任务页推进对话。不要把全部对象并列展示。

Rule 02默认不显示代码图谱

代码问答和制图是 Context，不是首页，也不是任务页的常驻右栏。

Rule 03减少面板，保留路径

Apple / Codex 质感来自安静、克制和清晰路由，不来自更多模块。

Screen 1Home: 继续或创建

显示下一个最该做的动作和最近项目。没有任务详情，没有图谱，没有证据墙。

→
Screen 2Project: 任务队列

只看当前项目任务列表、筛选、新建。代码理解是一个 Tab 或按需入口。

→
Screen 3Task: 连续对话

推送到 app-server 后进入任务对话页。这里才出现运行、暂停、完成和证据。

01

Home 不是 Dashboard

首页只回答一个问题：我现在该继续哪个任务？它不展示代码图谱、表格、设置、证据矩阵。

  • 首层：一个继续动作。
  • 收纳：项目列表与全局搜索。
  • 空态：引导选择项目或创建任务。

02

Project 只管理项目任务

项目页可以有任务、代码、会话三个入口，但首屏只服务当前入口。选中任务时进入任务页，不在项目页展开完整会话。

  • 首层：任务列表与新建任务。
  • 收纳：代码理解、会话、项目设置。
  • 错误态：只提示当前项目状态。

03

Task 才是连续对话

点击“打开任务”后才进入对话和状态推进。这里可以推送 app-server、继续 CLI、标记完成、查看证据，但不再同时展示项目全局列表。

  • 首层：消息、状态、composer。
  • 收纳：代码上下文、Git diff、附件证据。
  • 危险动作：取消、回滚进入确认抽屉。

证据锚点与这版删掉的东西

  • 产品定位：PRODUCT.md:7-13 证明 Zeus 同时做项目管理、真实代码扫描、AI CLI 执行、Git Diff 和远程控制，本版把它们拆进不同屏，而不是同屏堆叠。
  • 任务主线：DESIGN.md:16 明确 Task / Task Detail 以状态机、执行前检查、Runtime 会话、事件时间线、Git diff 和测试结果为中心。
  • 代码理解边界：DESIGN.md:17 说明 Code Map 是真实扫描图谱核心，本版保留为项目内入口和任务上下文，不放首页常驻。
