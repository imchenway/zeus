# TASK_20260701_007 Zeus 三屏极简动线设计稿

## 用户反馈

用户指出上一版仍然错误：越来越复杂，把所有东西都堆到一个页面。

## 纠偏结论

上一版的错误不是视觉不够高级，而是信息架构继续“驾驶舱化”。新的方向改成：

1. **Home：继续或创建**
   只显示当前最该继续的任务、最近项目和新任务入口。
2. **Project：任务队列**
   只显示当前项目的任务列表、筛选和新建任务；代码理解、会话、设置作为项目内入口。
3. **Task：连续对话**
   点击任务后进入对话页；推送 app-server、继续 CLI、暂停、标记完成、查看证据都在任务页内完成。

代码库问答和制图不再常驻首页，也不常驻任务页右侧，只作为项目内“代码理解”入口或任务页的按需上下文抽屉。

## 设计原则

- 一个页面，一个主动作。
- 默认不显示代码图谱。
- 减少面板，保留路径。
- 保留 Codex / Apple 式冷中性、低边界、系统字体、轻材质。
- 不复制外部品牌资产、字体、色名或 token。

## 证据锚点

- `PRODUCT.md:7-13`：Zeus 覆盖项目管理、真实代码扫描、AI CLI 执行、Git Diff、Telegram 远程控制。
- `DESIGN.md:5-17`：Zeus 是低疲劳 macOS 产品 UI；任务、代码图谱、会话各有自己的信息架构。
- `DESIGN.md:20-35`：偏冷中性色、避免纯白纯黑、卡片只用于分组、不做重复卡片网格。
- `DESIGN.md:145-190`：source-list、object-toolbar、composer、decision-rail、mode-rail、graph-canvas、drawer 的组件语义。

## 这版明确删除的内容

- 删除全屏跨项目任务 Lane。

- 删除 Working memory 面板。
