# TASK_20260701_008 Zeus 当前动线优化建议

## 本轮用户反馈

用户指出两点：

1. 之前的当前动线优化稿漏了代码页。
2. 设计稿有重网页感，但 Zeus 是一个软件。

## 本轮修正

本轮直接更新 `docs/TASK_20260701_008_Zeus当前动线优化建议.html`：

- 把展示方式从网页报告改成 macOS 软件规格板。
- 补齐代码页，与任务页、会话页并列成为当前动线的一环。
- 保留当前 Zeus 动线，不新增 Home，不把页面合并成驾驶舱。
- 明确三页分工：
  - 任务页：创建、筛选、打开任务详情，强调下一步和 AI 执行。
  - 会话页：承接任务，连续输入，推进状态。
  - 代码页：图谱画布主位，从节点、问答、源码入口创建或引用任务上下文。

## 代码页原则

代码页不是附录，也不是任务页右侧常驻面板。它在项目内保留独立页面：

- 图谱画布继续是主角。
- 图谱视图切换保留在顶部紧凑工具条。
- 检查器保留当前对象、打开源码、从节点创建任务、引用到任务。
- 图谱问答结果可创建任务或补充到任务上下文。
- 任务执行路径不常驻图谱，避免把任务页做复杂。

## 软件感修正

- 去掉大标题网页 Hero。
- 去掉营销式分区和重阴影。
- 使用 titlebar、outline rail、紧凑 pane、系统控件和桌面软件密度。
- 文案改成规格板语言，不写宣传语。
- 视觉仍遵循冷中性、低边界、系统字体和轻 accent。
