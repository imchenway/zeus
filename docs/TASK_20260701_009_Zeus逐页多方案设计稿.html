Settings · 三个可选方向

Settings：从顶部横排 Tab 改为可扫描的本机偏好面板

当前截图显示 7 个横向分类、大片空白和单组设置；优化重点是让设置像 macOS 偏好，而不是网页表单。

A

推荐：系统偏好式两栏

visual thesis：冷中性 macOS 偏好面板，低边界、低阴影、列表密度更高。

通用AI CLI / RuntimeTelegram安全与钥匙串Git 确认发布与更新
通用本机界面偏好，4 项
应用语言简体中文
深色/浅色模式跟随系统
桌面通知已启用
无未保存变更 · 保存只影响本机偏好

content plan：左侧分类 + 右侧当前组 + 底部变更条；首屏展示通用、Runtime、Telegram、安全的配置状态。

interaction thesis：设置修改只在底部出现“有未保存变更”；分类切换 120ms 淡入；reduced-motion 下直接切换。

  • 分类列表固定左侧
  • 状态摘要替代大空白
  • 保存条只在脏状态出现
B

搜索优先偏好

visual thesis：Raycast 式设置搜索，适合记得目标但不想找分类的高级用户。

搜索设置，例如 Telegram、语言、Git 确认
桌面通知通用 / 状态提醒已启用
AI CLI 适配器AI CLI / Runtime未配置
危险 Git 写操作确认Git 确认 / 安全必需
⌘K 跳转设置 · ↑↓ 选择 · Enter 打开

content plan：顶部搜索 + 最近改动 + 匹配设置行；分类只是结果过滤，不做页面主角。

interaction thesis：输入后结果即时过滤；键盘上下选择；reduced-motion 下取消结果位移。

  • 搜索框是首层主操作
  • 设置行保留路径面包屑
  • 高频设置可 pin 到顶部
C

系统健康式配置

visual thesis：设置变成“本机就绪度”，更像开发工具的 readiness 面板。

本机运行SQLite、app-server、项目路径正常
AI CLI等待本机登录后可运行任务
Telegram未启用远程控制
Git 确认写操作保留二次确认

content plan：本地运行、AI CLI、Telegram、安全、发布五组健康状态；每组只露下一步。

interaction thesis：未配置项给修复按钮；危险项进入确认抽屉；reduced-motion 下抽屉无位移。

  • 外部等待项更清楚
  • 适合首次配置
  • 常规偏好会退居二级

状态与可访问性口径

  • 默认：显示当前分类和只读状态摘要
  • 加载：分类 skeleton，不遮住左侧导航
  • 空/未配置：只显示等待项和修复按钮
  • 错误：说明影响范围，不暴露密钥

所有候选必须满足 24px 以上点击目标、focus-visible、非颜色状态表达、文本缩放可读与 reduced-motion 降级。

Sessions · 三个可选方向

Sessions：把会话页从居中空态改成持续执行工作台

当前截图里新对话居中且留白大，已有任务时上下文、环境和 composer 分散；优化目标是保持连续对话的工作流。

A

推荐：连续对话工作台

visual thesis：会话页像 ChatGPT/Codex 桌面工作区，但密度更像本机工具。

会话123 · 待开始分析图谱节点 · 运行中
123用户要求：123任务已创建 · 02:38
要求后续变更…
环境AI CLI 未配置Telegram 未启用Git 未检测

content plan：左会话列表、中央时间线、底部 composer、右侧项目/环境窄栏；新对话也不脱离框架。

interaction thesis：composer sticky；环境栏可折叠；发送后流式状态落在时间线。

  • 空态不再居中孤岛
  • 环境状态常驻但低噪
  • 任务状态操作贴近输入
B

任务交接 Deck

visual thesis：当前会话像一张正在推进的任务卡，但不是卡片堆叠。

任务 ZEU-000001待开始 · 可启动 AI
用户要求123事件任务创建 · 02:38
推送到 CLI 对话…

content plan：顶部任务状态带、中央用户要求/事件、底部输入；会话列表只做左侧索引。

interaction thesis：任务状态改变时状态带更新；危险操作 inline 确认；reduced-motion 下只变色。

  • 最适合任务驱动
  • 状态和下一步最清楚
  • 普通闲聊略显正式
C

Inbox 式会话流

visual thesis：把会话列表做成可批量清理的收件箱，适合会话很多时。

今天123 · 待开始分析 dependabot · 运行中图谱节点建议 · 暂停
预览最近事件、下一步和环境状态合并展示快速动作：继续 / 标记完成 / 归档
回复或创建跟进…

content plan：左侧分组列表占比增加，中央预览，右侧细节按需展开。

interaction thesis：快捷键归档/恢复；列表虚拟滚动；reduced-motion 下无滑动过渡。

  • 大量会话管理更强
  • 对单任务连续输入弱一些
  • 需要更好列表信息密度

状态与可访问性口径

  • 默认：时间线 + composer + 环境栏
  • 空：仍保留工作台，不用大 hero
  • external wait：AI CLI 未配置显示在 composer 附近
  • 错误：失败消息可重试并保留输入

所有候选必须满足 24px 以上点击目标、focus-visible、非颜色状态表达、文本缩放可读与 reduced-motion 降级。

Code Map · 三个可选方向

Code Map：图谱画布仍为主角，但把节点语义和任务衔接前置

当前图谱页方向正确，但节点标签重叠、右侧 inspector 信息块较厚、底部操作不够像图谱工作台。

A

推荐：画布优先 Inspector

visual thesis：大面积安静画布，节点与边直接承载来源，Inspector 是薄信息层。

系统架构图 · 模块图 · 依赖关系图 · 方法逻辑图

真实来源 · 8 节点 · 5 条聚合边

/Users/david/hypha/zeus · .github/dependabot.yml
节点详情文件 · 1-12

content plan：顶部图谱类型 + 扫描状态，中央画布，右侧节点来源/边/操作，底部 breadcrumb。

interaction thesis：节点 hover 只露来源和动作；选择后 inspector 更新；reduced-motion 下无缩放动画。

  • 图谱主角不变
  • 减少底部按钮噪声
  • 从节点创建任务更近
B

问答优先 Explorer

visual thesis：代码理解从自然语言入口开始，图谱是回答的证据面板。

问：这个节点影响哪些任务？
回答引用 3 个节点，2 条边，可加入任务上下文
证据来源文件、边权重、扫描时间

content plan：顶部问答输入，回答引用节点，画布展示相关子图，Inspector 显示证据。

interaction thesis：提问后子图高亮；引用可加入任务；reduced-motion 下只切换高亮。

  • 适合“我想知道 X”
  • 降低图谱学习成本
  • 全局浏览能力比 A 弱
C

影响路径 Trail

visual thesis：把图谱从全景变成影响路径，适合改动评估和任务生成。

入口文件模板行为任务建议
影响范围5 个节点 · 2 条边 · 1 个高置信路径
用当前路径创建任务…
路径证据每步可追溯到源码行

content plan：左侧路径轨道、中央局部链路、右侧影响范围和任务建议。

interaction thesis：路径节点可展开上下游；生成任务带默认上下文；reduced-motion 下轨道无位移。

  • 适合改动前分析
  • 任务衔接最强
  • 不适合首次全局理解

状态与可访问性口径

  • 默认：真实图谱 + inspector
  • 加载/扫描：画布 skeleton + 进度
  • 空：说明选择仓库/扫描下一步
  • 错误：保留上次成功图谱并提示重新扫描

所有候选必须满足 24px 以上点击目标、focus-visible、非颜色状态表达、文本缩放可读与 reduced-motion 降级。

Tasks · 三个可选方向

Tasks：任务列表要成为执行队列，不是只有一行的表格空场

当前任务页有空态、表格、创建弹窗和详情抽屉；优化目标是让下一步、AI 执行、证据和状态一眼可见。

A

推荐：执行队列表

visual thesis：Linear 式任务队列，表格清楚但不冷冰冰，下一步是第一列信息。

搜索任务 · 全部 · 标题 · 标签
任务编号下一步AI 执行证据
ZEU-000001可启动 AI未配置/Users/david/hypha/zeus
详情：事件、Runtime、完成/取消

content plan：顶部筛选 + 新任务，中央队列表，右侧轻量详情抽屉；空态也保留队列骨架。

interaction thesis：行选择打开详情；运行任务用 inline 状态；reduced-motion 下 drawer 直接出现。

  • 保留当前动线
  • 对一行和多行都成立
  • 实现成本最低
B

生命周期看板

visual thesis：按任务状态分列，适合拖动和批量梳理，但不做卡片墙。

待开始ZEU-000001 · 可启动 AI
运行中等待 Runtime 输出
阻塞AI CLI 未配置
完成保留证据

content plan：待开始、运行中、阻塞、已完成四列，每列紧凑 row；右侧详情仍按需。

interaction thesis：拖拽有键盘替代；状态改变弹出确认；reduced-motion 下无拖动动画。

  • 状态管理直观
  • 适合任务多
  • 单任务时信息略分散
C

焦点队列

visual thesis：一个当前任务 + 下方 backlog，强调“现在做什么”。

当前任务：ZEU-000001下一步：运行任务或补充要求
Backlog · 0完成后自动选择下一条
证据：事件、Git diff、Runtime 输出

content plan：顶部当前任务执行区，下面折叠 backlog，右侧证据/事件。

interaction thesis：开始任务后焦点区固定；完成后自动切下一条；reduced-motion 下无自动滚动。

  • 连续执行感最强
  • 最像个人工作台
  • 批量筛选弱于 A

状态与可访问性口径

  • 默认：队列表 + 详情
  • 空：显示队列骨架和创建入口
  • 运行中：inline Runtime 状态
  • 危险：取消/删除二次确认并审计

所有候选必须满足 24px 以上点击目标、focus-visible、非颜色状态表达、文本缩放可读与 reduced-motion 降级。
