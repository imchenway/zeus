# TASK_20260702_001 Zeus 设置页按钮与 Git 确认布局修复

## 任务背景

用户在 Zeus 设置页截图中标注了发布更新动作按钮边缘异常，并附带当前应用截图显示 Git 确认设置行内文案、目标信息与按钮互相挤压。当前仓库路径为 `/Users/david/hypha/zeus`。

## 仓库规范读取

- `AGENTS.md`：当前目录及上级未发现。
- `PROJECT-STYLE.md`：当前目录及上级未发现。
- `CODE-GUIDELINES.md`：当前目录及上级未发现。
- `DESIGN.md`：已读取 `/Users/david/hypha/zeus/DESIGN.md`。设置页应遵守 macOS 产品 UI、冷中性 token、低疲劳、控件状态清晰、卡片不堆叠、危险操作保留确认与审计。
- `PRODUCT.md`：通过 impeccable context loader 读取，Zeus 是本地优先 macOS AI 研发工作台，设置页不得伪造状态或弱化高风险确认。

## 调研与根因

### 现象

1. 发布与更新页的 `检查更新`、`GitHub Release`、`打开下载页` 一类动作按钮出现过强渐变与投影边缘，截图中按钮左上边缘显得像被额外高光或阴影污染。
2. Git 确认页的“危险操作必须确认”行在 680px 设置内容列内继续使用普通风险行三列，第二列和按钮列挤占空间，导致左侧标题/说明与右侧目标信息/按钮重叠。

### 根因

- `apps/desktop/src/renderer/styles.css:13396-13416` 的 Settings 动作按钮仍使用渐变与内阴影，和 Zeus Design Contract 的“安静产品控件、无厚阴影”不一致。
- `apps/desktop/src/renderer/App.tsx:9607-9610` 之前 Git 设置 pane 未带 `deep-settings-pane` 语义类，因此未进入深层设置的 780px 内容列与三列布局基线。
- `apps/desktop/src/renderer/styles.css:13437-13442` 新增前没有 Git 确认风险行的 Settings 专属列宽覆盖，普通风险行在设置详情列中容易压缩。

## 设计决策

- 不新增依赖，不改构建，不改数据模型。
- 复用现有 Settings reference shell 与深层设置列宽，避免新增独立布局系统。
- 按产品 UI 语义把动作按钮改为平面 token 控件：保留边框、hover、focus 与危险语义色，移除渐变和内阴影。
- Git 确认仍保持危险动作二次确认语义，只修复布局，不改变 Git 行为。

## 实现变更

- `apps/desktop/src/renderer/App.tsx:9607-9610`
  - Git 确认设置 pane 增加 `deep-settings-pane git-settings-pane`。
  - 目的：让 Git 确认页进入深层设置布局与内容宽度规则。
- `apps/desktop/src/renderer/styles.css:13396-13416`
  - Settings 页按钮改为 `--zeus-control-bg` / `--zeus-control-border`，移除 `linear-gradient` 与 `inset` 投影。
  - 危险动作按钮保留 `--zeus-danger-bg` 和危险边框，并显式 `box-shadow: none`。
- `apps/desktop/src/renderer/styles.css:13437-13442`
  - 新增 `.settings-reference-shell .git-settings-pane > .git-confirmation-risk-row` 专属列宽。
- `apps/desktop/test/app-shell-layout.test.tsx:2156-2188`
  - 回归测试锁定动作按钮不再使用渐变/内阴影。
  - 回归测试锁定 Git 设置 pane 必须具备深层设置类，且 Git 风险行有 Settings 专属列宽。
