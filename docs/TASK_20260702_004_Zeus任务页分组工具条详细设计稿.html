页面设计稿：Zeus 任务页分组工具条详细设计稿

本稿基于已选方向“分组工具条优先”。不再发散候选，因为方向已确认；这版只把选中方案展开为可落地的产品稿：首屏结构、尺寸口径、默认 / 加载 / 空 / 错误态、响应式和交互边界。

唯一核心目标

让任务页首屏成为可信的任务表格工作台：用户一眼能看到当前项目、筛选范围、任务状态和新建入口。

用户当前动作

搜索、切换状态、排序或创建任务；低频的标签、列设置、批量动作必须收起来，不抢主路径。

首层必须展示

项目任务对象、搜索、状态分段、新任务、视图摘要、表格默认 7 列和 44px 级别行密度。

收纳原则

标签和列设置进入右侧工具组；批量动作只在选择后强化；长详情仍进任务详情，不回到右栏停车场。

还原口径：这不是截图临摹版，而是生产版还原。目标是 85% 结构还原、75–80% 视觉像素还原，并严格沿用 Zeus 冷中性 macOS 产品 UI。

详细规格

实现时按这些口径收敛，不再靠截图反复调

布局尺寸

  • 工具条总高度目标：34px + 30px + 24px。
  • 表格行高目标：44–46px。
  • 表格与工具条间距：8px 内。
  • 内容区不为滚动条预留 72vh 空白。

信息优先级

  • 主操作：新任务。
  • 主过滤：搜索 + 状态分段。
  • 次过滤：排序 / 标签摘要。
  • 低频：批量、列、更多。

表格列

  • 默认保留 7 列：编码、任务、状态、AI、来源、证据、更新时间。
  • 右侧列不能视觉消失。
  • 宽列和扩展字段允许横向滚动。
  • 来源列只显示类型，不显示长路径。

收纳边界

  • 列设置弹层继续不被表格裁切。
  • 批量按钮默认低噪音，选中后进入批量栏。
  • 更多菜单只放低频设置，不放主路径。
  • 任务详情仍由详情抽屉承载。

视觉语气

  • 无 hero、无大卡片、无营销渐变。
  • 蓝色只给主按钮、焦点、选中态。
  • 表格线低对比，靠字重和间距分层。
  • 系统字体，不引入外部品牌字体。

交互节奏

  • 搜索即时过滤。
  • 状态分段一键切换。
  • 列设置 Escape / 外部点击关闭。
  • reduced-motion 下禁用位移动效。

页面状态

首屏不只画默认态，也要覆盖真实产品状态

默认

真实任务直接展示表格；不选中首行，不固定灰底。

加载

局部骨架替代表格，不遮住侧栏和工具条。

空

还没有任务，用“新任务”创建第一条。

空态仍是表格工作台，不回到半屏停车场。

错误

说明影响范围和恢复动作，不暴露堆栈、密钥或完整终端输出。

错误文案要能指导恢复，不伪造成功状态。

响应式

窄屏不把整张表缩成不可读缩略图

桌面

完整工具条 + 默认 7 列；横向滚动只服务宽列。

平板

状态分段可换行；低频动作保留在右侧或第二行。

窄屏

只展示核心字段，次要字段进行内展开或详情。

主路径页面流

每一步都对应首屏一个明确区域

定位对象

用户先看到 Zeus E2E · 任务，确认当前项目和任务范围。

搜索任务

搜索框在对象行中央，输入后表格即时过滤。

切换状态

常用状态用分段控件一键切换，减少下拉操作。

管理视图

排序与标签摘要在第二行，列和更多操作收纳到右侧。

扫描表格

默认列直接可读；右侧列不能被裁掉不可达。

进入详情

点击行打开任务详情抽屉，事件、AI 推送和 Git 证据不挤在首屏。

可访问性与实现验收

设计稿转实现时必须进入测试契约

键盘 / 读屏

  • 状态分段具备当前项语义。
  • 表格行继续保留 roving focus。
  • 列设置和更多菜单 Escape 关闭，并恢复焦点。
  • 批量状态变更通过 aria-live 告知。

实现验收

  • 任务页不出现旧内层左栏或右侧停车场。
  • 默认 7 列 header / row 都存在。
  • 非空列表有横向可达兜底但不撑高。
  • 工具条在 1180px / 760px 断点有清晰换行或收纳。
