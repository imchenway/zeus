页面设计稿：Zeus 任务页多场景原型设计稿

上一版只有一个首屏场景，确实缺了空任务、无结果、加载、错误、批量、列设置和新任务弹窗等原型。这里补成多场景设计稿，并把滚动策略改成硬规则：任务页生产界面不出现两个纵向滚动条；横向滚动只在表格列真的溢出时出现。

单滚动原则

直接回应“双滚动条”问题
1
纵向只交给一个容器生产页要么由工作区滚动，要么由列表区域滚动，不允许外层页面和表格内部同时纵向滚动。
2
默认态不展示假横向条7 个默认列能完整展示时，不画灰色横条；只有自定义宽列、扩展字段或窄宽度导致溢出时才出现横向兜底。
3
空态 / 无结果 / 加载 / 错误不制造滚动条这些状态没有宽表内容，不应出现横向条，也不应把列表撑成 72vh 之类的大空白。
4
列设置弹层不被表格裁切列面板属于工具条弹层层级，不依赖表格滚动容器；宽列选择后才让表格内容出现单一横向兜底。
5
仓库约束来源本轮未发现 AGENTS.md、PROJECT-STYLE.md、CODE-GUIDELINES.md；设计约束遵循 DESIGN.md 的任务页主工作区、真实状态和空态规则。
6
弹层不参与文档流列设置和新任务都必须是 overlay：覆盖当前表格，不把任务行往下顶，也不改变表格行高和滚动位置。

多场景原型地图

全部纵向展开，不用 tab 藏状态
默认2 个真实任务，默认 7 列完整可读。 新任务点击新任务后出现悬浮创建弹窗。 空任务没有任何任务时给第一步动作。 无结果筛选后为空，优先清除筛选。 加载局部骨架，不遮住工作台框架。 错误说明影响范围和恢复方式。 批量选择后出现批量动作条。 列设置弹层完整可见，宽列触发横向兜底。 窄屏收敛成任务卡片，不靠横拖阅读。

新任务弹窗点击新任务后覆盖当前表格，不挤压任务行；取消或创建后焦点返回按钮。

实现验收清单

把设计问题变成可测试约束
约束
必须满足
禁止回退
建议测试
默认列
7 列 header 和 row 都存在，证据、更新时间不能被隐藏成“剩余列”。
不能靠裁切右侧列来制造无滚动的错觉。
断言 header/cell 文本存在，并覆盖证据与更新时间。
滚动模型
非空短列表高度自适应；横向滚动只在列溢出时出现。
禁止恢复 block-size: min(72vh, 760px)、block-size: 100% 这类撑高滚动区。
样式断言：短列表不强制 72vh；宽列启用 overflow-x。
空态
0 任务、筛选无结果、加载、错误都要有独立文案和下一步。
不能用大空白或右栏停车场表示空态。
渲染测试分别覆盖 empty / no-results / loading / error。
新任务弹窗
点击新任务后弹窗覆盖当前工作区；创建 / 取消后焦点返回触发按钮。
不能把弹窗插入表格流导致任务行下移。
断言 dialog 存在、焦点 trap、取消恢复焦点、提交中禁用重复点击。
列设置
弹层完整可见，不被表格边界裁切；固定列不可隐藏；不把任务行往下撑。
不能把弹层放进会裁切的 table scroller，也不能参与表格文档流。
弹层定位测试 + 键盘焦点返回触发器。
窄屏
小窗口改为摘要卡或四列以下重点表，不把横拖作为唯一阅读路径。
不能让 body 出现不可控横向溢出。
390px 宽度无页面级横向溢出。

本轮对上一版的修正

明确哪些地方要改掉

应该保留

  • 分组工具条优先：搜索、状态、新任务是首层主路径。
  • 视图控制条：排序、标签、批量、列、更多在第二层低噪音呈现。
  • 默认 7 列完整渲染，任务页仍以表格工作台为主角。
  • 新任务和列设置都用悬浮 overlay，独立于表格滚动容器。

必须去掉

  • 默认列表底部的假灰色滚动条。
  • 外层页面滚动 + 内层列表纵向滚动的双滚动条。
  • 短列表为了把滚动条推到底部而撑出的空白高度。
  • 空态、无结果、加载、错误都只用默认场景含糊带过。
  • 把列设置或新任务弹窗插入文档流，导致任务行被往下顶。
