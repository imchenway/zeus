## 2026-07-02 追加返修：更多菜单误做成大表单

### 用户现场问题

- 现场截图显示点击 `更多` 后，右侧出现排序/标签的大块表单面板，覆盖表格头和空状态区域。
- 这违背了“分组工具条优先”的设计：`更多` 应是轻量菜单，而不是二级筛选表单，也不能把用户注意力从任务表格主路径拉走。

### 根因

- `TaskWorkspace.tsx` 中 `task-table-view-more-panel` 复用了 `task-filter-control-row` / `task-toolbar-select` / `task-toolbar-tags` 表单结构。
- `styles.css` 中该浮层宽度为 `min(320px, calc(100vw - 32px))`，并按表单行排布，因此在空状态/窄宽度下视觉上变成盖住表格的大面板。

### 修复（已被 2026-07-03 追加返修修正）

- 当时只把 `更多` 改为 220px 轻量浮层，并尝试把排序和标签压成小行。
- 这个口径仍然错误：它没有删除 `更多 = 排序 / 标签表单` 的信息架构。
- 2026-07-03 已追加修正：`更多` 只保留动作菜单，排序和标签不再出现在该浮层内。

## 2026-07-03 追加返修：More 动作菜单彻底移除排序 / 标签表单

### 用户现场问题

- 打包后的 Zeus 点击 `更多` 后仍然展示 `排序` 下拉框和 `标签` 输入框，形成右侧大块表单弹层。

- `更多` 只保留动作型菜单，当前为 `清除筛选` 与 `恢复默认列`。

## 2026-07-03 追加返修：无动作时禁止 More 死悬浮

### 现场问题

- 用户现场截图显示：默认空任务 / 无筛选 / 默认列状态下，点击 `更多` 后仍固定悬浮一个菜单。
- 菜单里 `清除筛选` 是禁用项，`恢复默认列` 在默认列状态下仍可见，形成无意义浮层。
- 这违反了任务页原型的低噪音原则：没有可执行动作时，不应该弹出 More 菜单，更不应该在表格和空态上方固定悬浮。

### 修复口径

- `更多` 只在存在真实可执行动作时可打开。
- 当前可执行动作只有两类：
  - 存在搜索 / 状态 / 标签筛选时，显示 `清除筛选`。
  - 列显隐、列顺序或列宽偏离默认时，显示 `恢复默认列`。
- 默认状态下：`更多` 按钮禁用，且不渲染 `task-table-view-more-panel`，因此不会再出现死悬浮。
- 若菜单打开后状态变回无可执行动作，组件会主动关闭菜单。
