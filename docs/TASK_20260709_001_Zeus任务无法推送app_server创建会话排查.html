故障排查：Zeus 任务当前未能推送本机 app-server 创建会话

本轮只读排查结论：仓库实现里任务运行 API 和 Runtime 会话创建链路存在，聚焦 API 测试通过；但当前本机进程探测没有 Zeus/Electron/Node app-server 监听，因此当前现场无法通过本机 app-server 创建会话。若用户 App 内仍报错，下一跳应抓 app-server 实时错误响应和 Runtime adapter 状态。

用户现场用户点击任务的“发送到对话/运行任务”；当前期待进入本地 Runtime 会话。
入口链路Renderer main.tsx 把 onRunTask 绑定到 client.runTask，再请求 /api/tasks/:taskId/run。
核心模块Local Server 的 /api/tasks/:taskId/run 会取任务与项目，然后调用 startTaskRuntimeSession。
外部依赖Runtime manager 最终 spawn 默认 adapter：codex exec <prompt>；需要 app-server 进程和 CLI 环境同时可用。
现象当前探测不到 Zeus app-server 监听；所以“推送到本机 app-server 创建会话”没有可达目标。
证据E1: ps/lsof 未发现 Zeus/Electron/Node app-server；E2: app.whenReady 才启动本地服务；E3: API 测试证明路由存在。
疑点最高可疑点 H1：桌面 App 当前未启动或已退出；H2：若 App 已启动，则可能是 Renderer 拿到旧 baseUrl/token 或 Runtime adapter 启动失败。
修法先启动/恢复 Zeus 桌面 App，使 Main 启动 startDesktopLocalServer；若 App 内仍失败，再抓具体 API 响应。
验证已验证：codex 在当前 shell 可见；任务 Runtime 控制聚焦测试通过。未验证：用户可视 App 内的实时错误响应。

结论

  • 已确认链路存在：桌面主进程启动本地 app-server，Renderer 的任务运行入口请求 /api/tasks/:taskId/run，Local Server 再创建 Runtime session。
  • 当前现场最高可疑点：这次探测时本机没有 Zeus app-server 进程/监听端口，所以任务没有可推送目标。
  • 不是代码里缺少“创建会话 API”：聚焦测试 packages/local-server/test/task-control-api.test.ts 已验证任务运行 API 能返回 201 和 runtimeSession.id。

现状链路

  • apps/desktop/src/main/main.ts:587-604：只有 Electron Main app.whenReady() 后才调用 startDesktopLocalServer()。
  • apps/desktop/src/main/localServerRuntime.ts:55-82：本地服务生成临时 token、启动 startZeusLocalServer，只把 baseUrl/apiToken 暴露给 Renderer。
  • apps/desktop/src/renderer/main.tsx:137-139：任务运行按钮最终调用 client.runTask(taskId)。
  • packages/local-server/src/index.ts:2136-2150：服务端 POST /api/tasks/:taskId/run 调用 startTaskRuntimeSession。
  • packages/local-server/src/index.ts:7051-7104：成功路径会创建 session、记录审计、发布 runtime.session.created。

当前断点

ps/lsof 探测结果：未发现 Zeus、@zeus、local-server、pnpm dev、Zeus Electron 或 Node app-server 进程；监听端口列表里也没有 Zeus 本地服务。 因此本轮只能判定“当前本机没有可用 app-server 目标”。

注意

  • 如果你肉眼确认 Zeus App 已打开，则这里存在“工具探测和用户现场不一致”，需要以 App 内实际错误响应/端口配置为准继续取证。
  • 若 app-server 存在但仍失败，下一类原因会转向 Runtime：并发上限、项目路径、adapter 命令、CLI 登录/权限。

已排除 / 已验证

  • CLI 当前 shell 可见：codex --version 返回 codex-cli 0.142.5。
  • API 聚焦测试通过：pnpm vitest run packages/local-server/test/task-control-api.test.ts -t "runs, pauses, continues, cancels, and retries" --reporter=verbose 结果 1 passed。
  • 测试覆盖点：packages/local-server/test/task-control-api.test.ts:293-309 断言 /api/tasks/:taskId/run 返回 201 且 runtimeSession.id 存在。

下一步取证

  • 先启动或恢复 Zeus 桌面 App；确认本机出现 127.0.0.1:<port> 的 Zeus local-server。
  • 在 App 内点同一个任务的“发送到对话/运行任务”，抓真实 API 错误：如果是连接失败，看 Main/app-server；如果是 400/409，看 Runtime adapter、cwd、并发或 Generic shell 确认。
  • 如需进入修复阶段，需要先拿到 App 内错误响应或本机日志；当前还不能把 H2/H3 写成根因。
