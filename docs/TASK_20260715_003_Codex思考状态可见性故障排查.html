A：视图层 turn 占位active turn 且当前 turn 尚无非用户可见 item 时，在消息流末尾显示“Codex 正在思考”；真实 item 到达即移除。不写 DB，不伪造 provider 事件。推荐，符合 Codex App 可见语义

C：只保留顶部状态维持当前远场反馈，无法回答“消息是否已开始处理”。排除
  • AC1：starting_turn / active_prework / active_final_answer 的首 item 空窗，在最新用户消息下方显示近场状态。
  • AC2：首个真实 reasoning、assistant、tool 或 request 内容出现后，通用占位立即消失，且不与现有空 assistant item 的 thinking 重复。

  • AC4：动态状态使用 role="status" / aria-live="polite"，reduced-motion 下无扫光或位移动画。
