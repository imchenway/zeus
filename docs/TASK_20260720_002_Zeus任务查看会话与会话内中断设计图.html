目标职责链

用户先从任务确认工作对象，再进入关联会话观察或控制当前执行。任务状态可以展示会话事实，但不能替代会话的运行控制。

任务目标、状态、证据
→
会话历史、上下文、输入
→
执行轮次运行、完成或中断

例外态：没有关联会话

任务详情不显示一个无法完成导航的“查看会话”按钮，也不伪装出 Runtime 控制。任务仍可通过“推送到模型”创建或选择会话。

当前实现事实

  • 现有任务详情根据任务 status=running 显示“暂停 Runtime”。
  • 任务级暂停接口会更新任务状态，并只停止传统 Runtime 管理器持有的运行中会话。
  • 截图中的任务显示“未启动 Runtime 会话”，因此现有按钮的控制对象并不清晰。

目标验收口径

  • 任务详情中反向搜索不到“暂停 Runtime / 继续 Runtime”。
  • 有真实关联会话时，“查看会话”能定位到准确会话；无会话时不显示。
  • 会话执行中可中断当前 turn；中断后 thread、历史消息和继续输入能力仍保留。
