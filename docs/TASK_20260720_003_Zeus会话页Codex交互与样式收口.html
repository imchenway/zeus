边界风险与后续动作
1. 用户消息底部大块空白气泡高度应只由消息内容决定。
消息 action rail 改为绝对定位去掉隐藏 footer 的 28px 最小高度；悬停或聚焦时再显示。
样式反向断言 + 正式包截图session-workspace.test.tsx 校验 position:absolute 且无 min-block-size。
通过气泡底部空白已消失。
保留可访问性按钮仍在 AX 树中，键盘聚焦会显示。
2. 命令输出样式杂乱命令多时不应把会话主线切碎。
单行命令摘要 + 可展开详情摘要只显示终端图标、命令预览、状态与复制；工作目录、耗时、退出码和输出在展开层。
正式包真实展开/收起AX 树确认 disclosure triangle；展开后显示完整元数据和输出。
通过默认密度明显降低。
不丢证据完整 stdout/stderr 仍可查看，没有删减 provider 事实。
3. 收发消息按钮按 Codex App 复刻用户与 Codex 消息需要不同的操作顺序和相同的轻量线性图标语言。
两套 28px action railCodex：复制、赞、踩、展开、时间；用户：时间、复制、编辑。改为与参考图一致的 1.7px 线性 SVG，保留选中、展开和复制反馈。
参考图同屏对比 + 正式包 AXdocs/evidence/zeus-assistant-actions-comparison-20260720.png 与 zeus-user-actions-comparison-20260720.png；AX 确认按钮顺序与 toggle 状态。
通过图标形状、顺序、灰度与时间位置均已收口。
反馈作用域赞/踩是当前消息的本地交互反馈，不伪造服务端持久化；复制由 Electron Main 写入并回读一致后才显示“已复制”。
4. 模型、推理、权限不可切换需要真实影响后续会话，不是只改本地文字。
实时能力选项 + 自定义上弹列表模型/推理选项来自 app-server 当前 generation；三类菜单统一固定在触发器上方，当前值始终留在下方可见；权限仍沿用真实 PATCH 链路。
状态链测试 + 正式包菜单截图121 项会话聚焦测试通过；docs/evidence/zeus-model-dropdown-packaged-20260720.jpeg 显示完整列表在触发器上方，AX 暴露真实 listbox/options。
通过空闲态可切换，忙碌态保持安全锁定。
切换时机steer_now 不支持中途换模型；只在新轮次发送时生效。
5. 右上角就绪图标粗糙状态要轻量、语义明确。
CSS 语义状态图标移除原始字符 · / ◌ / ×；就绪为环+内点，忙碌为旋转环，警告/错误使用伪元素。
无原始 glyph 的渲染断言正式包右上角已显示环形就绪状态，文本仍为“已就绪”。
通过与 Codex App 顶部密度一致。
减少动效系统开启 reduced motion 时保留静态语义。
6. 输入框选中时去除黑色边线聚焦不应让整个对话框突然变重。
聚焦态保持同一边框 token:focus-within 不再切换独立深色边框；键盘焦点继续由具体按钮的可访问 outline 承担。
反向搜索 + 正式包聚焦截图源码已无 session-composer-focus-line；docs/evidence/zeus-composer-focus-packaged-20260720.jpeg 中聚焦前后外框一致。
通过黑色焦点边线已移除。
不牺牲键盘可见性下拉选项和动作按钮仍有独立 focus-visible 指示。
7. 用户操作栏与下一条内容太挤操作栏不能与下一条消息正文争抢同一段垂直空间。
为绝对定位操作栏预留独立尾部间距有操作栏的消息增加 has-message-actions 语义类和 18px 尾部留白；按钮与气泡保持 5px 距离。
最终正式包截图docs/evidence/zeus-session-message-actions-final.png；正文、操作栏、下一条消息三层不再重叠。
通过间距已从偶然的 flex gap 变为显式布局约束。
不恢复气泡内空白留白位于消息外部，不重新撑高用户气泡。
8. 图标粗糙且复制疑似无效图标要接近 Codex 参考图，复制必须作用于真实系统剪贴板。
定制线性 SVG + 原生剪贴板桥复制、赞踩、展开、编辑统一 1.7px 圆角线条；Renderer 经 Preload IPC 调用 Electron clipboard.writeText，Main 随即 readText 校验。
真实按钮状态 + 写后回读最终包 AX 从“复制消息”切换为“已复制”；只有 Main 回读与目标文本一致才会进入成功态。
通过不再依赖 file:// 页面不稳定的 Clipboard API。
三层兜底原生桥不可用时才尝试浏览器 Clipboard API 与选区复制；失败保持原标签。
9. 编辑必须发生在原对话气泡内点击铅笔后不能把内容跳转到底部输入框。
气泡原位切换为编辑表单保留完整原文，自动聚焦并增高；提供取消和发送编辑内容；Escape 取消，Command/Ctrl+Enter 提交。
最终包原位编辑截图docs/evidence/zeus-session-inline-edit-final.png；AX 显示编辑文本域、取消、发送编辑内容均位于原用户消息容器。
通过编辑过程不离开原气泡。
失败保留草稿重新发送失败时编辑器不关闭，并给出文字错误提示。
10. 底部输入框应随内容自动增高多行内容不能继续挤在固定的单行高度中。
按 scrollHeight 自适应草稿、输入事件和窗口尺寸变化都会重新计算高度；最小 48px，最大为视口 34%，超过上限后才启用内部滚动。
纯函数边界测试 + 正式包截图resolveAutosizeTextareaHeight 覆盖最小值、中间值与上限；docs/evidence/zeus-session-composer-autogrow-final.png 显示 6 行内容完整展开。
通过控制行始终位于内容下方。
防止遮挡会话达到视口上限后保持固定高度并滚动，不无限推高输入框。
11. Command+A 应触发全选macOS 用户在输入区需要标准编辑快捷键。
补齐原生 Edit 菜单角色应用菜单新增 undo、redo、cut、copy、paste、selectAll 标准 role；输入框同时暴露 Meta+A / Control+A 快捷键语义。
最终包整段替换先输入 alpha beta gamma，按 Command+A 后输入 ZEUS_SELECT_ALL_OK，AX 最终值只剩新文本；菜单栏可见 Edit。
通过恢复 macOS 标准全选行为。
全应用受益使用 Electron 原生 role，不把快捷键硬编码到单个输入框。
12. 会话列表按 Codex App 扁平展示项目下不再显示任务分组和摘要；任务会话统一显示“任务编码：任务名称”。
项目级扁平时间序列项目会话保留自身标题；任务会话由真实任务编码和名称生成单行标题，同一任务的多会话允许标题相同。
组件渲染断言ProjectConversationTree.tsx 与 session-workspace.test.tsx 覆盖扁平顺序、无摘要、无任务组和键盘 roving tab。
实现并通过自动化验证列表语义与截图一致。
标题不是会话 ID相同标题通过内部 conversation id 独立选择，不伪造序号。
13. 蓝点只表示模型成功完成且尚未打开失败、中断、运行中和当前已打开会话都不能误显示完成蓝点。
持久化完成未读 + 幂等确认成功非临时轮次写入 completion_unread；选中会话通过独立 PUT 确认清除，不改 updated_at。
存储、协调器、REST、Renderer 四层测试覆盖成功/失败/中断、重启恢复、后台事件、当前会话抑制和重复确认。
实现并通过自动化验证蓝点拥有独立持久化语义。
活跃状态优先正在响应、等待批准或等待输入时隐藏旧蓝点，避免两个互斥状态同时出现。
14. 等待批准与需要用户输入使用截图状态审批为绿色胶囊，用户输入为蓝色胶囊；连接和响应完成前使用灰色旋转环。
拆分权威运行状态选中会话读取 controller；后台会话由持久化 pendingRequestKind 区分 approval 与 user input。
状态映射与 SSR 断言测试验证两种胶囊文案、旋转环、ready 无尾部状态，以及 reduced-motion 静态回退。
实现并通过自动化验证不再用模糊的“等待操作”。
失败保持显式错误和旧会话仍使用单独的低噪声图标/文字，不冒充完成状态。
15. 命令/文件审批复刻紧凑授权面板主操作为“允许一次”，下拉可选类似命令，右侧独立“拒绝”。
Phosphor 图标 + fail-closed 菜单终端/文件图标、上弹授权菜单、Escape/方向键/外部点击和拒绝优先焦点；只接受 provider 精确广告的 execpolicy amendment 对象并原样发送。
Renderer、REST、协调器、app-server manager 测试覆盖合法对象原样透传、规则可见、空数组/额外字段/规则篡改拒绝，以及文件审批仅一次授权。
实现并通过自动化验证MCP、permissions 和用户询问继续使用原有专用表单。
不扩权“允许类似命令”不是整会话放行；没有精确 provider 规则时绝不显示该选项。
