# Zeus 窗口与界面偏好持久化修复

## 任务目标

- Zeus 重新启动后恢复上次主窗口所在显示器、位置、尺寸、最大化和全屏状态。
- 显示器拔出、排列变化或分辨率变化时，窗口必须回到可见工作区，不能落在屏幕外。
- 任务表格的列显隐、顺序和列宽必须完整持久化。
- 搜索词、当前选中项、弹窗和批量选择等临时操作状态不跨启动恢复。

## 持久化边界

### Electron Main 窗口状态

窗口状态由 Electron Main 进程负责，写入 `app.getPath('userData')/main-window-state.json`。三屏复验后状态格式升级为 v2：

- `bounds`：上次正常窗口的 `x/y/width/height`。
- `display.id`：上次窗口所在显示器的 Electron ID。
- `display.label/internal/rotation/size/scaleFactor`：显示器指纹，用于 macOS 重新枚举显示器 ID 后找回同一块物理屏。
- `display.workArea`：保存时显示器的可见工作区，用于显示器重新排列后恢复相对位置。
- `isMaximized`、`isFullScreen`：原生窗口展示状态。

旧 v1 文件仍可直接读取；下一次真实用户移动、缩放或展示状态变化保存时自动写成 v2，不在读取阶段擅自改写用户文件。

恢复规则：

1. 依次按显示器 ID、完整指纹、部分指纹匹配；相同型号仍冲突时选择最接近上次工作区中心的显示器。
2. 目标屏暂未枚举时监听 `display-added` 和 `display-metrics-changed`，最多等待 2 秒；超时后使用原窗口尺寸在主显示器居中，展示后不再突然跳屏。
3. 原显示器仍存在时，按原工作区偏移恢复位置，并把窗口完整限制在当前工作区内。
4. 窗口构造时先设置一次边界，`ready-to-show` 前再次 `setBounds` 并验证真实落屏；落屏错误只允许纠正一次，随后才恢复最大化或全屏并展示。
5. 恢复阶段的 `move/resize` 等系统事件不允许写状态；进入 `active` 后，真实用户变化才使用防抖保存，关闭窗口时只刷新尚未落盘的真实变化。
6. 状态文件缺失或损坏时，使用 `1240×820` 默认尺寸在主显示器居中，不阻塞应用启动。

### Zeus 产品偏好

任务表格偏好继续使用 `app.shell.settings`：

- `visibleColumnKeys`：可见列。
- `columnOrder`：列顺序。
- `columnWidths`：每列的紧凑、标准或宽列档位。

Local Server 启动时会规范化旧列键、旧默认列顺序和非法列宽，并把迁移结果立即写回 `zeus.db`
，避免每次启动重复迁移。局部更新列显隐或顺序时保留既有列宽；显式传入空 `columnWidths` 时恢复默认列宽。

### 反向边界检查

- 搜索代码与存储结构后，未把任务搜索词、状态筛选、当前选中项、批量选择、弹窗或抽屉状态加入持久化模型。
- 窗口首次启动仍使用默认尺寸，但改为在主显示器工作区居中；已有有效状态时不再强制回主显示器。
- 非法、未知或已删除的列键和列宽不会原样写回；局部更新列显隐或顺序不会清空已有列宽。

## 未纳入范围

- 不持久化任务搜索词、筛选条件、当前选中任务、批量选择、临时弹窗或抽屉状态。
- 不修改 Git 历史，不执行 commit、push、merge 或 revert。
